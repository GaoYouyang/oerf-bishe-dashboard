# 来源与 DOI 审计记录

生成日期：2026-07-10

用途：开题和论文写作时，避免直接照抄 OERF 网站源码中的早期占位 DOI、旧题名或索引冲突。正式引用优先按出版社、DOI/Crossref、期刊页面；OERF 官网和交大主页用于确认课题组方向与代表性成果。

## 来源优先级

1. 出版社页面、DOI、Crossref 元数据。
2. arXiv 正文或 PDF，用于开放阅读和方法细节。
3. OERF 官网、OERF GitHub 源码、蔡老师交大主页，用于确认课题组方向、项目线索、代表性论文。
4. Semantic Scholar、ResearchGate、机构知识库，用于发现线索，不单独作为正式引用依据。

- 逐种子 mean superiority 暴露稳定性边界：K=4 为 `+3.45/+3.04/-1.31%`，K=6 为 `+3.65/+7.17/+5.11%`，K=8 为 `+1.43/-0.96/+2.60%`。seed collapse 后的小正均值不可替代 worst-seed 和两层 field/seed 统计。
- K=6 的优势主要来自 IID/noise OOD，thin-front family/joint OOD 对 FNO 仍有轻微退化。`audit_reprojection_rel_l2` 使用 synthetic clean held-out projection，只能称 retrospective clean Q_audit，不是实际 noisy camera 证据。
- v3b attention 子模块在 camera/angle 同步置换时不变；完整 FNO trunk 使用 canonical camera slots，因此不能声称整个网络对任意输入顺序严格 permutation invariant。下一版工作标签是 zero-initialized set adapter：初始严格等于 FNO，只在新 dev2 上学习小修正，并把 blind final fields 与独立 forward 一次性锁定。

## 2026-07-11 T16 v3c K=6 dev2 负结果补记

- `demo_t16_operator/results/v3c_k6_dev2_pilot/` 比较 base FNO、同 checkpoint 追加 12 epochs 的 continued FNO 和冻结底座的 zero-init adapter。三个分叉的 checkpoint hashes 逐种子一致，adapter 初始输出差和 12 epochs 后底座参数漂移均为 0。A/B 同 batch order、loss、learning rate、optimizer 与 validation rule，但只匹配追加 epochs，不匹配 FLOPs。

## 2026-07-11 v3d 最近邻查重补记

- NeurIPS 2025 官方论文页确认 **F-Adapter** 系统研究了 Fourier-based large operator models 的 PEFT，并报告普通 LoRA 明显弱于 adapter；其方法按频率复杂度为低/高频分配不同 adapter 容量。因此“frequency-adaptive adapter for FNO”本身不能作为本项目创新。来源：`https://proceedings.neurips.cc/paper_files/paper/2025/hash/a12e362d89d4e0b40760f839f91550ee-Abstract-Conference.html`。
- PMLR 官方页确认 ACML 2025 **R2-FFNO** 对 FFNO spectral kernels 做 reduced-rank factorization，并讨论 rank 饱和和高频 augmentation。因此“低秩 spectral kernel”本身不能作为创新。来源：`https://proceedings.mlr.press/v304/chou26a.html`。
- TMLR/OpenReview 的 **MG-TFNO** 已用全局 tensor factorization 表示 Fourier-domain parameters；NeurIPS 2023 **GINO** 已用 point-cloud/SDF 与 graph-FNO 处理可变物理域。因此本项目必须把参数压缩、物理域 geometry 和 BOST 相机/光线 acquisition geometry 三者分开。来源：`https://openreview.net/forum?id=oFqHIkw8sd`、`https://proceedings.neurips.cc/paper_files/paper/2023/hash/70518ea42831f02afc3a2828993935ad-Abstract-Conference.html`。
- ICML 2024 **FourierFT** 已在一般 PEFT 中用少量 Fourier coefficients 表示参数更新，进一步排除“把权重更新放到频域”这一宽泛新颖性。来源：`https://proceedings.mlr.press/v235/gao24o.html`。
- 当前只保留工作假设：用 angle/ray coverage/mask/noise/calibration 等相机级统计条件化 F-Adapter 的频带容量或低秩系数。它必须在 saturated FNO、LoRA、vanilla adapter、F-Adapter、last-block 以及 correct/shuffled/constant/static geometry controls 下通过，才能进入 blind final。完整门槛见 `v3d_prior_art_and_novelty_gate.md`。

## 2026-07-11 v3d FNO validation-plateau 审计补记

- `demo_t16_operator/results/v3d_fno_saturation_audit/` 沿用 v3c 的 K=6 input contract、validation-locked ridge、24-epoch base schedule、loss 与三模型种子。随后以固定 12-epoch blocks、learning rate 0.001 继续；若 block 最佳 checkpoint 没有至少 `1e-5` validation 绝对改善，则回退到上一个 checkpoint。
- plateau 选择只读取 validation：连续末端两个 blocks 必须同时满足 mean-seed 相对改善不超过 0.5%、任一 seed 不超过 1.0%。若后续重新改善，之前的临时平台不算 plateau。dev2 test 与 clean Q_audit 在判决后才计算。
- 红队复核后 validation 改为每个三维场等权；到 96 epochs 仍未 plateau。mean validation L2 为 `0.16646→0.11746`，末个 block 的 mean/max-seed 改善仍为 `2.813%/2.959%`；21 个 seed-checkpoints 均接受新 checkpoint，没有发生 validation 回退。
- `96 epochs` 是 validation-selected checkpoints 串接的累计标签；每个 12-epoch block 都重新初始化 AdamW 与 cosine scheduler，不携带 optimizer state，因此不能描述为一次无缝连续训练。
- dev2 后置诊断相对 epoch 24 的 domain-equal field superiority 为 `+16.540% [15.611%,17.452%]`；clean Q_audit L2 为 `0.41978→0.34211`，三个 seeds 与四个来源域均未反向。统计单位仍是 128 个已查看 linear synthetic dev2 fields，不能把 2,688 per-seed rows 当作独立物理样本。
- 当时科学判决为 `V3D_FNO_VALIDATION_PLATEAU_NOT_REACHED_BY_MAX_EPOCH`：暂停新分支并继续审计 optimizer/schedule；后续固定 epoch 判决见下一节。blind final 仍未生成 seed 或开启。
- `v3d_geometry_data_manifest.md` 将另一闸门显式化：真实 geometry 若固定且没有不同 K、缺失相机、标定漂移或跨装置需求，acquisition-conditioned 方法直接 No-Go。

- PSU 正式论文/开放正文说明每次 test acquisition 含 2000 张 flow-off 和

  `https://arxiv.org/html/2508.17120v2`。

  能降低 background-dependent systematic error；反复平均同一静态 reference

  `https://proceedings.mlr.press/v130/borovitskiy21a.html`、
  `https://proceedings.mlr.press/v151/nikitin22a.html`。因此“在 detector graph
  上加 covariance”本身不作为创新；必须由 BOST flow-off acquisition、
  held-out camera、inverse/adjoint 与 front-reconstruction 闭环提供独立贡献。
- 本地 DG-CovGate 只给出真实 detector geometry 上的 synthetic acquisition

## 2026-07-17 covariance-weighted inverse 下游重建核验

- BOS uncertainty 主证据补齐：Rajendran et al.,
  `10.1088/1361-6501/ab60c8` 使用 displacement uncertainty 与稀疏 Poisson
  operators 传播 density uncertainty，并报告 sharp density changes 会把
  uncertainty 传播到视场其他位置。OSTI accepted manuscript 可公开阅读。
- correlated-noise inverse 的新颖性边界补齐：Generalized least-squares CT
  已明确联合 detector blur 与 correlated noise model；因此“对测量做 covariance
  whitening/GLS”不能作为本项目独立新意。BOST 贡献必须来自真实 flow-off
  acquisition、detector geometry、同调用 inverse 与 reacting-front/tail-risk
  验证。
- 新增 `DetectorCovarianceWhitening` 与 `WhitenedMeasurementOperator`：
  component-IID、diagonal、graph、amplitude 与 low-rank drift 都可组成显式
  `L/L^T`；定向测试验证 factorized/dense whitening 一致、detector adjoint
  identity 与 BOST wrapper `K F + K A^T` 调用合同。
- 单种子 smoke 在 graph noise 下给出 DG-CovGate/oracle 中位 field gain
  `+2.567%/+2.611%`，但只作为机制筛查。随后在结果打开前提交
  `f061acb`，冻结 16 个全新 calibration/field/noise replicates。
- fresh multi-seed 主结果：DG-CovGate mean gain `+1.178%`，replicate-cluster
  95% CI `[+0.786%, +1.571%]`，front-F1 mean `+0.01225`；但 p10
  `-1.029%`、>1% harm `10.94%`，未过预注册 `-0.5%/10%` 门，判 NO-GO。
- Oracle covariance 的 mean/p10/harm 与 DG-CovGate 几乎相同，说明本轮尾部
  主要不是 R=50 estimator error，而是 whitened objective、固定四步 early
  stopping 与 IID 条件下选择的 Sobolev preconditioner 之间的耦合。annular
  kernel 是最明确失败族。
- 下一步只做 post-open covariance-conditioned preconditioner / whitening
  tempering 诊断；若存在 Pareto 改善，再冻结全新种子。未通过 deterministic
  baseline 前，不训练 learned selector，不声称优于 NeRIF/FNO/DeepONet。
