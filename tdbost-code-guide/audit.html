Reproducibility Boundary

为什么不能克隆后就直接训练？

科研代码公开不等于当前电脑已经具备数据、环境、二进制和作者服务器路径。下面把“阅读没有问题”和“已经复现成功”严格分开。

当前判断：注释版适合阅读，但公开快照尚未形成 Mac 上的完整可运行包。 中文注释没有修复任何问题；23 个 Python/PYI 文件去除注释后与师兄原版执行语法树一致。

已确认的直接阻塞点

位置现象实际影响

主训练路径需要作者确认的行为

1

活跃优化器与学习率衰减对象不一致

实际 step 的是 optimizergapp，学习率衰减和上采样后重建的却是 optimizerrho。需要确认当前公开快照是否遗漏了同步更新。

2

随机采样数表达式没有写回变量

N_samples + torch.randint(...) 只计算新值，没有赋值，因此不会改变 N_samples。

3

位移校正分支被乘以零

dp = self.app_regressor(...) * 0 会让该输出和对应梯度为零。需要确认这是消融设置还是调试遗留。

4

一个光线方向函数重复使用 x 列

get_ray_directions() 中 i 和 j 都取 point[...,0]。常见写法的 j 会取第 2 列，但必须结合作者的数据坐标约定确认。

5

PDE 联合训练分支可能引用未建立变量

modelU 不为 None 时只建立 optimizerrho，后续仍调用 optimizergapp。默认主线不触发，但进阶分支可能报错。

建议的复现顺序

  1. 向师兄确认论文对应版本、数据字段、相机坐标、物理单位、许可证和标准训练命令。
  2. 把服务器绝对路径移出 import 顶层，先做到所有模块可以安全 import。
  3. 建立与作者一致的 CUDA 环境，用极小伪数据跑 1 batch / 1 iteration。
  4. 记录活跃优化器中每组参数的学习率、梯度和更新量。
  5. 再接完整数据、检查点与论文指标；代码能跑不等于结果正确。
