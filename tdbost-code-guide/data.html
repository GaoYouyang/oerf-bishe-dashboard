Data Contract

一条 BOS 训练数据到底是什么？

代码不会直接把一张普通 RGB 图片交给模型。它把背景点、相机位姿、时间和二维偏移整理成一条条带物理意义的光线样本。

JSON 中的一帧

{
  "file_path": "./train/datatensor_170.npy",
  "rotation": 8,
  "time": 0.5,
  "transform_matrix": [[...], [...], [...], [...]]
}
  • file_path：该帧的 NumPy 数组路径。
  • rotation：相机或观察角度编号。
  • time：归一化时间。
  • transform_matrix：4×4 相机位姿矩阵。

NumPy 每一行怎样被解释

列代码中的角色需要向师兄确认
0背景点横坐标像素单位、坐标原点
1背景点纵坐标正方向、坐标原点
2第一位移通道是否为 u/x 位移及其单位
3第二位移通道是否为 v/y 位移及其单位

从像素到训练样本

1

像素 + 相机内参

像素减去主点，再除以焦距，得到相机坐标系中的观察方向。

2

相机外参

用 4×4 位姿矩阵把方向和相机中心变到世界坐标。

3

拼成训练输入

rays = [origin_x, origin_y, origin_z, direction_x, direction_y, direction_z]。

4

附上监督信息

同一行的后两列作为真实 u/v 位移，再附上 time 和 w2c。

变量名提醒 代码中的 all_rgbs 沿用了 NeRF 风格命名，但在这条 BOS 仿真主线中实际保存的是二维偏移，不是普通 RGB 颜色。
