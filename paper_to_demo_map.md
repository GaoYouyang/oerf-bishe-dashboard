# 核心论文到 Demo / 选题映射表

用途：把“读论文”直接转成“做什么实验、写哪一章、问师兄什么”。这份表是开题报告和见何远哲时最该随手打开的文献导航。

## 总判断

当前最稳的毕业设计主线是：

```text
BOS/BOST 基础
  -> NeRIF / neural field 重构
  -> 视角数、噪声、容量、真实数据接口鲁棒性
  -> 按师兄需求升级到 PIV-BOST 或 4D BOST 子问题
```

文献不要只按时间读，要按“它能支撑哪个 demo / 哪个论文章节”读。

## 读论文的最小动作

读完每篇 P0/P1 论文后，不写长摘要，必须交付以下四件之一：

1. 一个公式块：观测量、未知量、forward model、loss。
2. 一个参数表：视角数、噪声、分辨率、rank、采样数。
3. 一个 demo 改动：新增一个变量扫描或一个指标。
4. 一个给师兄的问题：能直接影响 A/B/C 主线或 E 副线的定题路径。
