主线可复现性审计

用途：把 NeRIF、PIV-BOST、4D BOST、公开数据和可用代码分开看清楚。这里的目标不是“证明哪个方向一定能复现”，而是给你和何远哲师兄一个诚实的起点：哪些材料可以公开读，哪些必须等组内数据/许可，哪些能先做成本科可交付 demo。

总判断

最稳路线仍是“少视角 BOST 的神经隐式折射率场重构与鲁棒性分析”。NeRIF 是主轴，PIV-BOST 是有真实同步数据时的强应用升级，4D BOST 是低秩时序/数据质量报告的高阶拆分。公开数据和开源代码只负责让你在没有组内数据前也能跑通 pipeline，不替代 OERF 火焰实验结论。

公开代码与许可边界

资源 当前结论 怎么用 不要怎么用
Hyz617/TDBOST 结构参考 本地已核 README、configs、TDmodel、dataloader、render、run.py 等入口；未见 LICENSE。 学习工程组织、配置层、张量模型分块、数据加载和可视化输出。 不要直接复制代码到公开仓库；不要把 README 的 open-source 描述当作法律许可。
photon / PIV-BOS synthetic image generation GPL-3.0 适合读代码和生成合成 BOS/PIV 图像，但依赖较重。 把 ray tracing、density/refractive-index gradient、synthetic image pair 和误差分析拆成简化 Python demo。 不要一开始完整移植 CUDA/C++ 依赖；不要把合成图当真实 OERF 火焰结论。
OpenPIV Python / OpenPIV BOS 练手工具链 适合图像互相关、矢量场后处理和 BOS 位移估计入门。 服务 PIV-BOST 的 image-pair benchmark，先跑传统 baseline，再谈神经光流或补偿。 不要把 OpenPIV 说成能直接完成 NeRIF 或 3D BOST tomography。
PIV 折射误差 / 光学畸变校正文献组 3 篇开放 PDF 已缓存 包括 PIV in inhomogeneous refractive-index fields、PIV optical distortion correction 和 PIVlab/OpenPIV 支撑文献。 用于把 M3A 从“速度场 toy”推进到 image-layer dewarping、ray tracing back-projection、互相关位移和 vector-field correction 的对照实验。 它们只能证明公开前史和工程接口，不等于已经验证了组内 simultaneous PIV-BOST 实验。
Neural implicit tomography / NIRT 方法邻居 IOP open PDF 已缓存 Kelly and Thurow 2024 提供 coordinate neural representation tomography for flow diagnostics。 用于比较 NeRIF 与更通用的 neural implicit tomography：投影监督、field representation、传统 tomography baseline 和 sparse-view 行为。 不要把它写成 BOST 专用方法；它是 related work 和实现 sanity check。
event_based_bos 旁支启发 官方 TPAMI 代码/数据路线；与蔡组 event-triggered BOS 旁支相互呼应。 学习 event/frame 数据结构、HDF5、config、optical-flow 评估和高时间分辨可视化叙事。 除非组里明确要事件相机，不建议把它变成本科主线。
NeDF + AIP Figshare supplement 正式 DOI + CC BY supplement NeDF 是 NeRIF 的外部竞品；补充材料含 metrics、training hyperparameters 和 network structures。 用于做 NeRIF vs NeDF/NRIP 表示方式对照表，提取指标定义和训练参数，不必完整复现全部实验。 不要把 NeDF 改成主线；它是评审问“外部 neural BOS 方法有哪些”时的对照材料。
Single View Refractive Index Tomography with Neural Fields / brandonyzhao/rtnf arXiv PDF + supplement 已缓存MIT code CVPR 2024 方法邻居，处理 neural field 中的折射率层析和 curved-ray tracing。 只借 forward model、single-view ambiguity、source prior 和代码组织，帮助检查 NeRIF toy 里的光线模型假设。 不要写成 BOST baseline、不要替代 OERF/何远哲主线；它没有提供火焰 BOST 多视角实验结论。

三档毕业设计落地方案

A 档：最稳开题

题目：面向少视角 BOST 的神经隐式折射率场重构与鲁棒性分析。

  • 依赖：NeRIF 论文、synthetic phantom、Open BOS manifest。
  • 交付：M0/M1/M2 + 自动图表 + 参数扫描表。
  • 风险：真实火焰数据未到时只能做公开 benchmark 和 toy，需要在报告里诚实标注。

B 档：组内数据增强

题目：基于真实 BOST 样例的 NeRIF 重构质量控制与参数选择。

  • 依赖：九视角样例、mask、view geometry、baseline 或参考切片。
  • 交付：data loader、bad-view report、重投影验证、resolution/noise/mask ablation。
  • 风险：数据公开边界和单位/标定字段必须提前问清。

C 档：应用升级

题目：基于 BOST 折射率场的 PIV 速度测量误差补偿预研。

  • 依赖：同步 PIV-BOST 数据，或合成粒子图像 + 简化折射场。
  • 交付：raw image / displacement / velocity 三层补偿对照。
  • 风险：没有同步数据时不能声称验证了真实 PIV-BOST。

D 档：高阶探索

题目：四维 BOST 的低秩时序先验与可视化评估。

  • 依赖：4D BOST 正文、TDBOST 结构参考、M3B low-rank toy。
  • 交付：rank-memory-time 表、temporal consistency、frame-count ablation。
  • 风险：完整 TOG 复现不适合本科承诺，只能拆子模块。
