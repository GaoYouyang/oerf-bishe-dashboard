光流/位移估计到 PIV-BOST 的桥梁

这页回答 PIV-BOST 和 BOST 前处理里一个很容易混淆的问题：Horn-Schunck、Lucas-Kanade、Farneback、FlowNet、PWC-Net、RAFT 这些方法负责估计图像位移或速度代理；它们不是折射率场重构算法本身，但会决定 BOST/NeRIF 看到的观测质量。

总判断

给师兄看的说法：如果我同步 PIV-BOST，应该把任务拆成 image-pair displacement、uncertainty/confidence、refractive-index compensation 或 reconstruction 三层。光流文献服务第一层和第二层；NeRIF/PIV-BOST/4D BOST 仍然是折射率场、PIV 速度补偿和时空重构主线。

这个区分对本科题目很重要。若没有 OERF 的完整九视角 BOST 数据，也可以先做一个可验证的图像层工具：用合成 BOS/PIV 图像比较 OpenPIV 互相关、Farneback、RAFT-PIV 或 RAFT-style optical flow 的位移误差，再把误差传播到一个简化折射率/密度重建 toy。这样题目贴实验需求，又不会把“深度光流”包装成完整 BOST。

九篇怎么读

论文 该抽什么 接到 PIV-BOST / BOST 的位置 本地/在线入口
Horn-Schunck 1981
Determining Optical Flow		 亮度守恒、全局平滑正则、aperture problem 和变分迭代。 解释为什么位移场需要正则、mask 和 confidence；适合写 BOS 光流 baseline 的数学根。 DOI
Lucas-Kanade 1981
Iterative Image Registration		 局部窗口最小二乘、图像梯度、Gauss-Newton 更新。 连接 PIV interrogation window、BOS 局部位移和 feature tracking；窗口大小会直接影响偏差/方差。 DBLP
Farneback 2003
Polynomial Expansion		 二次多项式邻域模型、dense displacement、实用 CPU baseline。 适合先做 BOS 图像 pair 的快速 dense optical-flow 对照；OpenCV 路线容易复现。 本地 PDF / DOI
Brox et al. 2004
Theory for Warping		 coarse-to-fine、warping、robust data term、gradient constancy。 说明强位移/大形变下为什么需要金字塔和 warp；这也是后续深度光流的共同骨架。 作者 PDF / DOI
Barron/Fleet/Beauchemin 1994
Performance of Optical Flow Techniques		 endpoint error、angular error、benchmark sequence、failure modes。 给 M3A/M3C 位移估计 demo 的评价指标；不要只放漂亮矢量图。 作者 PDF / DOI
FlowNet 2015 FlowNetS/C、correlation layer、synthetic training data。 深度光流起点；可用于解释 RAFT-PIV 不是“凭空的 AI”，而是图像位移估计范式变化。 arXiv PDF / DOI
FlowNet2 2017 stacked network、small displacement、intermediate warping、训练 schedule。 说明深度光流要解决真实小位移和细节，不只是堆 CNN。 arXiv PDF / DOI
PWC-Net 2018 pyramid、warping、cost volume 三件套。 把传统多尺度思想和 CNN 特征匹配连起来；比直接读 RAFT 更容易理解。 arXiv PDF / DOI
RAFT 2020 all-pairs 4D correlation volume、recurrent update、upsampling、泛化。 RAFT-PIV 的直接母体；如果做 neural optical flow 对照，这篇要和 RAFT-PIV 一起读。 arXiv PDF / DOI

本科可落地的实验拆法

层级 可做内容 需要的数据/代码 对何远哲方向的价值
D0 sanity check 用一张背景点图合成已知位移，比较 LK/Farneback/互相关的 endpoint error。 Python + OpenCV + OpenPIV；自己合成小位移和噪声。 快速建立“位移估计误差会污染重构”的直觉。
D1 BOS image-pair benchmark 用公开或组内脱敏 BOS 图像 pair 输出 displacement、mask、confidence 和残差图。 OpenPIV/BOS、Farneback、可选 RAFT/RAFT-PIV；需要问师兄图像格式和标定参数。 直接贴近 BOST/NeRIF 的观测前处理。
D2 PIV-BOST image-layer compensation 把 BOST 折射位移或折射率场转成 PIV 粒子图像去畸变，再跑 PIV 位移/速度。 一对原始 PIV 图像、同步 BOS/BOST 位移或折射率场、OpenPIV baseline。 最贴何远哲 PIV-BOST，但依赖组内真实数据和补偿接口。
D3 neural optical-flow ablation 对比 Farneback/OpenPIV/RAFT-PIV 在噪声、粒子密度、背景纹理、强折射下的表现。 公开 PIV 数据、RAFT-PIV 数据或合成图像；最好有 GPU。 可作为 PIV-BOST 的工程支线，不替代 NeRIF/4D BOST。

问何远哲师兄的问题

  1. 组内 BOST 当前位移估计主要用 cross-correlation、optical flow、wOFA，还是自写方法？
  2. PIV-BOST 的补偿发生在 raw image、calibration/disparity、correlation map，还是 exported velocity field 上？
  3. 是否能给一组脱敏 image pair：reference/background、distorted background、PIV particle pair、BOST reconstruction 或位移场？
  4. 如果没有真实数据，本科阶段用合成 BOS/PIV 图像做 displacement benchmark，师兄是否认可？
  5. 最终评价更看重 endpoint error、velocity error、reprojection residual，还是折射率/密度场误差？

写作边界

不要写成：光流算法可以直接重构折射率场。

应该写成：光流/互相关从 BOS 或 PIV 图像中估计二维位移场；BOST/NeRIF 再把多视角位移观测、几何标定和折射率物理模型结合起来做三维/四维重构。

这条边界能保护你的题目：你可以大胆学习 RAFT/PWC/FlowNet，但毕业设计主线仍然贴 OERF 的实验诊断问题，而不是变成普通计算机视觉 optical flow benchmark。
