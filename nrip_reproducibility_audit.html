NRIP 可复现性审计

NRIP 是 NeRIF 最值得正面对照的 2026 年外部方法邻居。这里把论文证据、公开仓库、许可边界、真实运行门槛和本科可做的 clean-room 消融实验放到同一页，目的是回答一个实际问题：哪些思想值得学，哪些代码不能直接搬，第一月到底该做什么。

结论先行

NRIP 很适合成为你的 NeRIF 外部竞品 + 消融设计来源，但不适合直接成为本科主代码。最稳的做法是独立实现一个缩小版 coordinate-field baseline，在同一 forward model、同一 view/noise/ray budget 下比较 Fourier、简化 hash-like encoding、mask 和 gradient formulation；等何远哲确认组内需求和可用数据后，再决定是否接真实 BOST。

方法链条

通俗地说，NRIP 不把三维空间切成一大堆固定体素，而是让一个小网络回答“这个坐标的折射率是多少”。相机位移不是标签场，而是沿光线积累折射率梯度后得到的观测，因此训练本质上是一个可微的物理反演。

1 几何与 phantom相机位姿、ROI、背景板、折射率真值或实验位移。
2 BOS 图像参考图 / 扰动图、有限孔径与模糊、光流位移。
3 坐标编码无编码、Fourier 或多分辨率 hash 表示。
4 折射率 primitive小型 MLP 输出连续 n(x,y,z)，而非直接存体素。
5 光线积分沿 ray 采样并积分 ∇n，预测多视角 deflection。
6 误差与验证位移 loss、held-out view、场误差、时间和显存。

一个很重要的复现警告

README 示例把训练设为 10,000 iterations、学习率 2e-2、每步 256 rays、每条 ray 最多 256 samples；但 argparse 默认值是 30,000、1e-2、4096、1024，代码中的 encoding 选择也需要手动确认。也就是说，“运行默认命令”不等于“复现论文配置”。你的实验登记表必须保存完整 config、commit、随机种子和输入数据版本。

NeRIF 与 NRIP 到底差在哪

维度何远哲 NeRIFNRIP本科可验证的问题
网络输出multi-head 同时输出 n 与 ∇n 的三个分量。以 n(x,y,z) 为唯一 primitive，再用 automatic / discrete gradient 形成观测。单输出是否更稳定；多头一致性约束是否换来更好精度。
编码sinusoidal frequency encoding；7 层、每层 256，第四层 skip。重点比较 Fourier 与 multiresolution hash；紧凑 MLP。固定参数量和 ray budget 后，频率编码、hash-like encoding 的细节/噪声 trade-off。
梯度约束自动微分梯度、网络直接输出梯度及其投影位移的一致性。比较 automatic、central-difference 和 hybrid；论文倾向用 discrete gradient 驱动稳定收敛。二阶导图、噪声放大、训练稳定性和运行成本。
空间约束神经场与采样策略；真实九视角 Bunsen flame。2D mask 选 ray，3D mask 硬约束有效体积，并配 occupancy sampling。mask 阈值错误时，artifact、漏检和 runtime 如何变化。
最公平的比较同一 synthetic n 场、同一 camera geometry、同一 optical-flow noise、同一 train/held-out views、同一训练预算。不要拿不同数据、不同网格或不同 GPU 的单个最终误差下结论。

第一月 clean-room 路线

  1. Week 1：物理闭环。 自己生成 3D Gaussian refractive-index phantom，用直线近轴模型得到多视角 deflection；确认单位、坐标和有限差分梯度。
  2. Week 2：最小神经场。 用小型 Fourier-coordinate MLP 只输出 n，先在 32³ 或 64³ 采样网格上跑通；设置一视角 held-out，不追求论文分辨率。
  3. Week 3：两项关键消融。 做 gradient formulation 与 2D/3D mask；每个实验保存 config、seed、曲线、field slice、reprojection 和失败原因。
  4. Week 4：组内对齐。 把 5/7/9-view、noise 和 ray-budget 结果压成一页，拿给何远哲确认是否值得接真实数据、Open BOS dataset 或组内 NeRIF baseline。

软件与硬件判断

当前 Mac 上

适合做 NumPy/PyTorch 的 32³-64³ toy、Fourier encoding、central difference、mask 和指标。不要把 CUDA raymarching、MEX 编译和整仓复现设成第一月验收条件。

能启动先做小体积

有 NVIDIA GPU 后

再尝试 128³ 以上、hash encoding、occupancy grid、更多 ray/step 和真实数据。先问组内推荐 CUDA/PyTorch 版本，避免环境调试吞掉研究时间。

第二阶段需环境确认

最容易写错的地方

折射率不是温度

BOS 直接测的是折射率梯度相关位移。温度换算依赖 Gladstone-Dale、状态方程和组分假设；复杂燃烧场里不能把两者无条件等同。

重投影低不等于场真

少视角反问题可能有多个场解释相近位移。必须保留 synthetic ground truth、held-out view、mask sensitivity 和失败案例。

公开不等于开源许可

GitHub 能访问只赋予查看/分叉平台权限，不自动赋予复制、修改或再发布许可。你的主代码应独立实现并记录思想来源。

带给何远哲的 7 个问题

  1. 组内 NeRIF 当前更想改 encoding、gradient consistency、mask，还是 ray sampling？
  2. NRIP 应作为正式 baseline、related work，还是只作消融灵感？
  3. 九视角真实数据能否提供一帧 reference/deformed images、位移场、geometry 和 mask？
  4. 真实系统最常见失败是 optical flow、标定、mask、强梯度，还是网络收敛？
  5. 评价更看重 refractive-index field、density/temperature、held-out reprojection，还是 downstream PIV compensation？
  6. 本科第一月交付 5/7/9-view + noise + mask 报告，是否比完整复现 NRIP 更有用？
  7. 组内是否有可用 NVIDIA 环境与推荐的 CUDA/PyTorch 版本？

我的真实建议

把 NRIP 当作一把非常好的“实验设计尺子”，不要当作你必须搬过来的工程包。你的毕业设计最有价值的创新不是声称又发明一个网络，而是用统一物理 forward model 和真实质量控制，把 NeRIF-style 与 NRIP-style 的有效边界测清楚，并让结果能无缝接入何远哲的九视角数据。
