XMU WebVPN 与私有论文库边界

这页给自己和师兄说明：学校 VPN 可以帮助个人阅读受限文献，但公开 GitHub Pages 只放合法可公开链接和开放 PDF。最后更新：2026-07-09。

公开 PDF 跳转库 公开缓存边界审计 受限全文访问队列 选题路线深挖矩阵 学习日志模板

核心原则

公开网站只缓存能确认开放许可或开放来源的 PDF。通过 XMU WebVPN、图书馆订阅、出版社机构访问拿到的 PDF 只进本机 `private_library/`，不提交到 GitHub，不做公开跳转页。

为什么即使“只是给师兄看”也不上传订阅 PDF

这个工作台当前通过 GitHub Pages 发布，页面地址可以被浏览器直接访问；即使仓库本身不主动传播，Pages 上的文件仍可能被公开读取、缓存或转发。仓库是否设置为 private 也不能作为分发订阅 PDF 的依据，因为期刊平台和学校数据库许可通常限制全文再发布。学校 VPN/图书馆订阅 PDF 适合个人学习、页码定位和本机精读笔记，不适合作为网页附件发布。公开页会保留 DOI、出版社入口、私有队列、阅读目标和我们自己的分析；PDF 本体放在本机 private_library/pdfs/。

2026-07-09 实测：无人值守下载边界

当前尝试用 WebVPN 代理后的 Springer PDF 地址直连 PIV-BOST 论文时，服务器返回的是 /fingerprint 指纹/验证页，而不是 PDF。后续不在无人值守状态下硬闯验证；如果需要补全文，白天由用户在浏览器里完成验证码或学校授权流程，把 PDF 放进 private_library/inbox/，再由本地同步脚本匹配到私有论文库。

下载后如何真正编入你的论文库

如果 PDF 是通过 WebVPN、学校订阅或个人账号下载的，把它当作“本地私有全文”：保存到 private_library/inbox/ 后运行 python3 private_library/tools/sync_private_library.py。如果浏览器已经把文件放进系统下载目录，可以运行 python3 private_library/tools/sync_private_library.py --scan-downloads --max-scan-files 20，脚本会只读取最近下载的候选 PDF，匹配成功后复制到 private_library/pdfs/，并更新本机 private_library/index.html 与状态报告。

这个流程不会联网、不会上传、不会把受限 PDF 写进公开 paper_library/。公开网页只同步“已在私有库/待下载/出版社入口/精读目标”等状态，方便你发给师兄确认研究路线。

WebVPN 实测工作流

可用路径

  • 进入 XMU WebVPN 后打开厦门大学知识资源港。
  • 在数据库导航里按数据库首字母筛选，例如 `S` 找 SpringerLink / ScienceDirect，`A` 找 AIP / ACM。
  • 优先使用 WebVPN 代理后的“直达”按钮，不走 CARSI 或统一身份认证。
  • 进入出版社站内后，再用 DOI、文章题名或期刊名搜索论文。

无人值守规则

  • 遇到验证码、人机验证、二维码、统一身份认证或购买页，停止该篇下载尝试。
  • 把条目标为“需白天手动协助”，继续做公开来源检索和网页整理。
  • 不保存账号密码到网页、仓库或论文元数据。
  • 下载到本地的受限 PDF 只用于个人学习和页码定位。

公开网页可以写什么

可以公开

  • 题名、作者、年份、期刊、DOI。
  • 官方出版社入口、arXiv、NASA、OSTI、机构库、作者开放稿。
  • 自己写的摘要、路线图、选题建议、学习计划。
  • 开放许可 PDF 的本地缓存链接。

只放本地私有

  • 学校订阅下载的出版社 PDF。
  • 需要登录、机构权限或购买才能访问的 PDF。
  • 只允许个人研究学习、不允许再分发的 accepted manuscript。
  • 组内未公开数据、代码、标定参数和实验图片。

需要师兄确认

  • 真实 BOST/PIV-BOST/4D 数据能否出现在本科论文。
  • 组内代码能否复用、截图或描述。
  • 未发表实验图是否能作为内部答辩材料。
  • 本科论文公开版和内部版是否需要分离。

给师兄的一句话说明

这个网页中的公开 PDF 只来自可公开缓存来源；通过学校 VPN 读取的受限全文只用于本人学习，不上传公开仓库。若后续涉及组内真实数据或未公开结果，会单独按课题组要求处理。

本页不包含任何账号、密码或受限 PDF 链接。公开版本只用于研究计划沟通和资料边界说明。
