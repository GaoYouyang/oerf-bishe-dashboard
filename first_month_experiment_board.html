第一月总目标

工作重心已经转到师兄提出的“三维重建 + 算子学习”。T16 的 checkpoint、三种子消融、18-run reliability 筛查和 3-seed dual v1 均已完成；M3B 又完成 864 格跨形态无真值容量选择，证明 support/spectrum/temporal 多特征能把固定 rank 3 的 mean regret 从 1.561% 降到 0.252%。第一月剩余任务改为：让师兄审核 inverse/evolution、support/query 和 temporal capacity-control 是否对齐，随后只补独立专家、物理锚定修正、leave-one-geometry-out 与组内 geometry/data 接口。NeRIF 作为 forward physics 与 refinement。

四周节奏

第 1 周：审核当前 checkpoint

把已完成的 168-sample FNO/U-Net/lift 证据拿给师兄，先锁定任务定义。

  • 展示 IID、view/noise/joint/family OOD 与 95% CI。
  • 重点解释三视角 field/held-out 分叉和 family OOD 失败。
  • 确认输入、输出物理量、geometry、数据权限与无真值指标。
交付：审核 brief + 4 张图 + 7 个决策问题。

第 2 周：因果消融（已完成）

三个优化种子已经把 output parameterization、BOST loss 与参数量因素拆开。

  • Residual/absolute 在 IID 与 3-view OOD 发生稳定翻转。
  • 带 reprojection 的五域均值方向更好，但三种子区间仍宽。
  • 43k Residual FNO 在 30/30 seed-domain 指标单元优于 49k U-Net。
已交付：4 组 ablation + paired seed CI + 3 张图 + 审核 brief。

第 3 周：简单 gate 筛查（已完成）

六种 FNO 对照验证单标量 physics-lift gate 能否吸收少视角翻转；答案是否定的。

  • 固定 gate 在 view/joint OOD 明显变差。
  • metadata gate 的 field oracle regret 只从 3.10% 变为 3.08%。
  • observed residual 有诊断性，但平铺 3D 通道使训练退化。
已交付：18 runs + 4 张图 + oracle regret + 可复跑脚本。

第 4 周：双分支 v1 已完成，v2 开始

已实现共享双输出、view dropout、support/query 和 query router；并保留了 router collapse。现在转向 support-consistent v2。

  • 补独立/低共享专家与等容量 ensemble，防止把容量写成路由收益。
  • 以 support-fit `w_S` 为必须击败基线，只学受限 `delta_w` 或 nullspace residual。
  • 加入 identifiability 与 risk-coverage；无法超过 support-fit 就停止 neural routing。
  • 独立高分辨率与真实 geometry 接口随后进入第二月。
已交付 dual v1、闭式强基线和塌缩证据；待交付独立专家 + anchored/nullspace 消融。

降级规则

如果一个月内拿不到真实数据，不代表项目失败。保底路线是 A 主线：synthetic BOST + traditional baseline + NeRIF-style coordinate representation + view/noise/capacity scan + open-data manifest dry run。B/C/F 是升级，不是毕业生存条件。
