一句话定题建议

毕业设计题目优先定为 “面向少视角背景纹影层析的神经隐式折射率场重构与鲁棒性分析”。第一月先做 synthetic BOST + 传统 baseline + 简化 NeRIF；如果何远哲师兄给真实 BOST/PIV-BOST/4D 数据，再把同一套 pipeline 升级成真实数据接口、位移质量报告、PIV 误差补偿或 4D 低秩时序子模块。

A 路线执行包

最小毕业版本

  • 自建 3D 折射率 phantom：Gaussian blob、jet-like plume、flame-like shell 至少三类。
  • 生成 5/7/9/13 视角 BOS-like displacement 或 line-integral projection。
  • 传统 baseline 至少包含 FBP/SART/LSQR/TV 中一种；写清它只是数学对照，不等同真实 BOST 全物理模型。
  • 简化 NeRIF：coordinate MLP 或 Fourier-feature MLP，输入坐标输出折射率/密度标量。
  • 扫 view count、noise、mask、field capacity、positional encoding，并输出固定格式图表。

论文贡献写法

  • 不是声称提出新大模型，而是给 OERF/He 主线做可复现 baseline 和鲁棒性图谱。
  • 回答“什么时候神经隐式场比体素/正则化 baseline 更稳，什么时候不值得用”。
  • 把 synthetic -> open BOS dataset -> OERF 样例数据的迁移接口设计清楚。
  • 把失败案例写出来：坏视角、mask 错、几何误差、噪声过大、网络容量不足。

B/C/F 升级规则

B：PIV-BOST

只有在拿到 PIV 图像对、PIV 参数或速度场时升级。若只有最终速度场，就做 velocity-field error propagation；若有 raw particle image，才做 image-layer dewarping。

C：4D BOST

只拆低秩时序、rank scan、temporal consistency、可视化和数据清洗。不要承诺本科完整复现 ACM TOG 4D BOST。

数据请求模板

向师兄要数据时不要笼统说“给点数据”

更好的说法是：我想先接一份最小样例，验证 loader、几何、mask、位移和重投影检查是否正确；不需要完整数据，也不需要公开权限。

BOST/NeRIF 最小样例

  • 每个视角的 reference / deformed background 或已提取 displacement field。
  • view geometry：相机位置、视线方向、像素尺寸、背景距离、放大率或等效几何参数。
  • mask/ROI、单位说明、坐标系定义、flow-off/flow-on 对应关系。
  • 一张参考切片、旧 baseline 结果或重投影误差口径。

PIV-BOST / 4D 升级样例

  • PIV：raw image pair、dt、pixel-to-mm、window/overlap、已有 velocity field。
  • BOST：同步帧编号、折射率/梯度/位移场、PIV 平面位置。
  • 4D：3-10 帧小片段、帧率、view order、同步信息、已有 rank/重构参数。
  • 权限：哪些图可以写进本科论文，哪些只能组内汇报。

放弃与降级规则

  • 如果必须完整搭实验台才能开始，降级为 synthetic/open data + 数据接口工具。
  • 如果真实 PIV-BOST 只有最终速度场，没有 raw image，不承诺 image-layer dewarping。
  • 如果 4D BOST 代码/数据不可用，不承诺完整复现，只做低秩时序 toy 和指标工具。
  • 如果某条 OERF 方向很强但不接 He/BOST 数据，例如计算光谱仪、等离子体、金属颗粒机理，就放到背景或备选，不抢主线。
  • 如果外部论文 PDF 只能通过 WebVPN/订阅拿到，只进入本机私有库，公开网页保留 DOI/出版社入口。
