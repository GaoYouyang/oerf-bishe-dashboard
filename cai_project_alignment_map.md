# 蔡伟伟公开项目语言到本科毕设题目的对齐地图

生成日期：2026-07-06

用途：把蔡伟伟老师交大主页、OERF 官网和何远哲论文线索翻译成 Gao Youyang 可以用于开题、约师兄、约导师的题目语言。这里严格区分三层：公开事实、合理推断、需要向何远哲确认的事项。

## 1. 公开事实锚点

### 1.1 蔡老师主页列出的研究方向

蔡老师交大中文主页的公开研究方向可以压缩成五组关键词：

| 公开方向 | 对 BOST/NeRIF 毕设的含义 | 你的表达方式 |
| --- | --- | --- |
| Agent for Science | 课题组已经把科研自动化作为公开方向，但它不能替代物理问题 | 可作为“参数扫描、实验记录和自动报告”的辅助模块 |
| 航发、高超飞行器内外流诊断、实验流体力学、燃烧诊断 | 真实需求是极端流动/反应流的非接触诊断 | 题目应强调“实验流体力学中的光学测量与重构”，不要只说 AI |
| 计算成像：光场、线性/非线性层析、全息成像 | BOST、NeRIF、4D BOST 都属于计算成像和层析反演 | 用“计算成像 + 背景纹影层析 + 神经隐式表示”定位主线 |
| 微型计算光谱仪、高速快照式光谱相机、散射光计算 | 这是强旁支，算法思想相邻，但不直接贴何远哲 | 写作中可作为课题组图谱，不建议主动改主线 |
| 机器学习在流动显示和燃烧诊断中的应用 | 允许使用 ML，但应服务物理诊断 | 说“物理一致的神经重构”，不要说“端到端黑箱网络” |

### 1.2 蔡老师主页列出的近期项目线索

| 公开项目线索 | 可提取的需求关键词 | 对你最有用的翻译 |
| --- | --- | --- |
| 先进航空发动机非接触式测量，数据同化相关项目 | 非接触测量、实验数据、模型融合、误差传播 | BOST/NeRIF 输出应整理成可用于后续数据同化的结构化物理场 |
| 数据融合实验流体力学创新研究群体 | 多源数据融合、可复现实验工具、物理约束反演 | 把题目写成“面向数据融合实验流体力学的 BOST 神经重构与验证” |
| 单相机内窥背景纹影层析测量火焰折射率/密度/温度 | BOST、折射率场、密度场、温度场、内窥/多视角几何 | 这是何远哲 NeRIF、PIV-BOST、4D BOST 的直接前史 |
| 非定常火焰碳烟光学层析 | time-resolved tomography、多物理场、4D 测量 | 可以支撑 4D BOST 的背景，不宜一开始转碳烟主线 |
| 高光谱吸收层析火焰温度场 | 吸收光谱、非线性层析、温度反演 | 用来补反问题和 optical tomography 背景 |
| 燃烧室三维火焰和温度重构 | 3D reconstruction、实验图像到物理量 | 证明课题组长期需求不是单篇论文，而是持续的三维重构能力 |

### 1.3 主页最新论文给出的方向排序

主页“流体光学诊断及颗粒燃烧方向”中已经出现 4D BOST、stereo PIV-BOST、Computational Flow Visualization、碳烟 LII、颗粒燃烧、数字全息等条目。对你而言，排序应为：

1. **主线第一层：NeRIF / BOST 三维折射率场重构。**
2. **主线第二层：PIV-BOST 折射补偿。**
3. **主线第三层：4D BOST 的低秩/时序子问题。**
4. **旁支一：Computational Flow Visualization，用来写“大背景”和方法论。**
5. **旁支二：数字全息/颗粒燃烧，只在师兄明确给图像数据时转向。**
6. **旁支三：计算光谱/光计算，只作为“课题组全图谱”背景。**

## 2. 和何远哲方向的重合度

| 何远哲相关方向 | 对应的公开项目语言 | 重合度 | 本科可落地版本 |
| --- | --- | --- | --- |
| NeRIF | 背景纹影层析、计算成像、机器学习流动显示 | 最高 | 少视角 BOST 的神经隐式折射率场重构与鲁棒性分析 |
| PIV-BOST | 航发/高超/燃烧诊断中的非接触测量和折射误差补偿 | 高 | 用 BOST-style 折射场估计 PIV 速度测量误差，先做 2D toy 和误差传播 |
| 4D BOST | 非定常流动诊断、数据融合实验流体力学、多帧高速测量 | 高但风险大 | 做低秩时序先验 toy、rank 扫描和时序一致性指标 |
| Computational Flow Visualization | 光学方法 + 计算方法揭示复杂流动隐含性质 | 中高 | 把论文第 1 章写成“光学观测不等于物理场，必须通过计算重构” |
| Agent for Science | 智能体全自动科研 | 中 | 做参数扫描、失败案例索引、自动图表报告，作为项目工程化加分项 |
| 数字全息颗粒测量 | 颗粒燃烧、全息成像、三维定位 | 中低 | 若转向，则做颗粒检测/3D 定位/轨迹连接工具，不和 NeRIF 混成一个大题 |
| 微型计算光谱 | 计算光谱、硬件编码、反演 | 低 | 只作为备选；除非导师明确安排，不建议主动离开 He 主线 |

## 3. 推荐题目命名

### 3.1 首选正式题目

> 面向少视角背景纹影层析的神经隐式折射率场重构与鲁棒性分析

为什么它最稳：

- 包含 BOST 和 NeRIF 主线。
- “少视角”和“鲁棒性”适合本科生做系统实验，不要求完整复现 4D BOST。
- 能自然接入视角数、噪声、采样、坐标编码、真实数据迁移。
- 如果何远哲给真实数据，可以升级；如果不给，也有合成和开源数据保底。

### 3.2 面向导师项目语言的题目

> 面向数据融合实验流体力学的背景纹影层析神经重构方法研究

适合对蔡老师说。它把你的题目挂到“数据融合实验流体力学”和“非接触测量”上，但仍以 BOST 为核心。

### 3.3 面向何远哲沟通的题目

> BOST/NeRIF 可复现重构、误差分析与真实数据接口

适合对何师兄说。它更务实，强调你能交付代码、图、指标和数据接口。

### 3.4 PIV-BOST 升级题目

> 面向 PIV-BOST 同步测量的折射率误差补偿与速度场误差传播分析

适合在拿到同步 PIV-BOST 数据时升级。没有真实数据时，只能先做 toy，不应作为唯一承诺。

### 3.5 4D BOST 升级题目

> 面向四维背景纹影层析的低秩时序先验与重构一致性分析

适合在何远哲明确希望你接 4D 子模块时使用。这个题目听起来高级，但风险更大，应限定为低秩/时序一致性子问题。

## 4. 最值得向何远哲确认的 12 个问题

1. 组里现在最急的是 NeRIF 复现、PIV-BOST 补偿、4D BOST 子模块，还是实验数据整理工具？
2. NeRIF 论文中的真实数据是否能给一个匿名最小样例？
3. 如果真实图像不能给，能否给相机几何、背景位移场和参考重构体？
4. 组里传统 baseline 更认可哪一种：SIRT、Landweber、Tikhonov、UBOST，还是已有 MATLAB/Python 程序？
5. 真实 BOST 数据的最小单位是什么：原始背景图、扰动图、位移场、相机参数、mask、参考体？
6. BOST 数据的坐标单位、折射率/密度/温度换算链条是否已经固定？
7. PIV-BOST 数据是 2D PIV、stereo-PIV，还是只适合做误差传播讨论？
8. 4D BOST 的数据规模和显存压力大概是多少？本科生能否只做低秩 toy 或可视化工具？
9. 论文/开题中哪些图可以公开展示，哪些只能内部汇报？
10. 如果我做自动报告和参数扫描，师兄希望输出哪些图表和指标？
11. 这个题目更适合写成“算法复现与改进”，还是“实验数据处理工具”？
12. 最终毕设评价更看重可运行代码、真实数据结果、理论推导，还是完整文档和图表？

## 5. 不要踩的表达坑

| 不建议这样说 | 建议这样说 |
| --- | --- |
| 我想做 AI for flow | 我想做物理一致的 BOST/NeRIF 折射率场重构 |
| 我想复现 4D BOST 全文 | 我想先拆出低秩时序先验和重构一致性子问题 |
| 我可以同时做 NeRIF、PIV、4D、光谱仪、全息 | 我先保住 NeRIF/BOST 主线，根据数据条件升级一条支线 |
| 没有真实数据就做不了 | 我先用合成/开源数据建立 pipeline，真实数据用于迁移验证 |
| 我要发一篇很强的论文 | 我先交付稳定代码、参数扫描、误差边界和可复现图表 |

## 6. 开题报告中可直接使用的定位段落

本课题拟面向极端反应流与实验流体力学中的非接触光学诊断需求，围绕背景纹影层析技术中的三维折射率场重构问题，研究神经隐式表示、物理投影一致性约束和多视角观测数据融合方法。课题组公开研究方向覆盖航发/高超飞行器内外流诊断、计算成像、机器学习流动显示与数据融合实验流体力学；近期工作中，NeRIF、PIV-BOST 和 4D BOST 分别对应三维折射率场重构、速度测量折射补偿和四维时空重构。基于本科阶段可落地性，本课题优先建立合成与开源数据上的 BOST 重构闭环，并以视角数、噪声水平和采样策略为变量开展鲁棒性分析；若获得课题组真实样例数据，则进一步开展真实数据迁移或 PIV-BOST 误差传播分析。

## 7. 最现实的一年路线

| 阶段 | 时间 | 目标 | 对应课题组需求 |
| --- | --- | --- | --- |
| 预研 1 | 第 1-4 周 | 流体、BOS/BOST、反问题、PyTorch 基础补课 | 能读懂何远哲论文和实验变量 |
| 预研 2 | 第 5-8 周 | 合成 BOST forward model、传统 baseline、指标和可视化 | 建立非接触诊断到物理场的最小闭环 |
| 预研 3 | 第 9-12 周 | NeRIF-style coordinate field、视角/噪声/容量扫描 | 对齐机器学习流动显示和计算成像 |
| 开题后 1 | 第 1-2 月 | 形成稳定代码仓库和论文第 1-3 章 | 可复现实验工具链 |
| 开题后 2 | 第 3-5 月 | 做少视角、噪声、采样、mask、坐标编码系统实验 | 数据融合实验流体力学中的误差分析 |
| 开题后 3 | 第 6-8 月 | 真实 BOST 或 PIV-BOST 数据迁移；若无数据则做 open BOS benchmark | 连接课题组真实诊断数据 |
| 收尾 | 第 9-12 月 | 论文、PPT、代码整理、图表复现和答辩问答 | 可交付、可复核、可继续扩展 |

## 8. 我的真实建议

1. **不要为了显得高级而把主线写成 4D BOST。** 4D BOST 很亮眼，但本科毕设最容易失控。你可以把它作为第 4 章的升级实验或第 5 章展望。
2. **最稳的创新点不是发明新网络，而是把物理链条做扎实。** 从折射率场、折射率梯度、背景位移、重投影误差到误差传播，每一环都要能解释。
3. **你真正能给何师兄减负的地方是工程化和系统实验。** 很多博士生论文里的代码和数据流程并不总是本科友好；你若能把参数扫描、图表和 manifest 做清楚，会很有价值。
4. **如果能拿真实数据，优先要“最小样例”，不要一开始要完整数据集。** 最小样例包括 1 组背景/扰动图或位移场、相机几何、mask、单位说明和参考输出。
5. **开题时要把“无真实数据”写成风险，而不是致命缺陷。** 你的保底路线是合成数据 + open BOS/TBOS + 可验证 forward model；真实数据是升级项。
6. **和蔡老师沟通时，用项目语言；和何师兄沟通时，用交付语言。** 对导师说非接触测量、数据融合、计算成像；对师兄说数据格式、baseline、图、指标、下一周交付。
