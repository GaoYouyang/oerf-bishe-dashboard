# v5e-v5f 研究学习日志（2026-07-16）

## 1. 这两轮在做什么

本轮是在已打开的 v5c 数据上继续做机制诊断和方法开发，不是新数据上的确认性实验。原始结果见：

- [`demo_t16_operator/results/v5e_covariance_whitening_diagnostic/report.json`](../demo_t16_operator/results/v5e_covariance_whitening_diagnostic/report.json)
- [`demo_t16_operator/results/v5e_covariance_whitening_diagnostic/paired_whitening_summary.csv`](../demo_t16_operator/results/v5e_covariance_whitening_diagnostic/paired_whitening_summary.csv)
- [`demo_t16_operator/results/v5f_dual_regularization_postopen/report.json`](../demo_t16_operator/results/v5f_dual_regularization_postopen/report.json)
- [`demo_t16_operator/results/v5f_dual_regularization_postopen/method_summary.csv`](../demo_t16_operator/results/v5f_dual_regularization_postopen/method_summary.csv)
- [`demo_t16_operator/results/v5f_outer_audit_postopen_diagnosis/diagnosis.json`](../demo_t16_operator/results/v5f_outer_audit_postopen_diagnosis/diagnosis.json)

v5e 比较了对角 whitening 和 exact covariance whitening，并尝试用残差的有效自由度修正 Morozov 目标。v5f 把“校准正则化尺度”和“重建场的正则化方法”分开：先用 diagonal、DF-corrected、target 1.0 的规则校准半径，再分别用 GCV 和 UPRE 做场重建。

这里的 exact covariance 不是从真实相机重复测量得到的可部署估计，而是使用 clean synthetic truth 得到的 oracle。因此它适合回答“如果协方差完全知道，机制会不会明显改善”，不能直接证明真实相机上已经解决了协方差问题。

## 2. v5e：完整协方差几乎没有改善

把对角 whitening 换成 exact covariance whitening 后，最近邻 bank match 的变化很小：

| 方法 | 对角 whitening | exact covariance | 变化 |
|---|---:|---:|---:|
| GCV | 3/6 | 2/6 | -1 |
| UPRE | 2/6 | 2/6 | 0 |
| Morozov | 0/6 | 1/6 | +1 |
| residual-DF corrected Morozov | 4/6 | 4/6 | 0 |

所以不能说“使用完整协方差就成功了”。它只说明，在这组已经打开的 synthetic 数据和当前实现中，主要问题并没有被 whitening 单独消除。尤其要记住：exact covariance 本身来自 oracle，真实实验并不能免费得到它。

残差-DF corrected、target 1.0 的规则是 v5e 里比较稳定的候选：达到 4/6，选择半径 0 次落在边界，删去一个 fold 后的半径稳定性为 0.792。可是 exact covariance 版本的 outer validation MSE 仍约为 172.29；这个数相对于其他开发结果仍然很大，不能因为 4/6 和“没有边界”就宣布它赢了。它更像是一个较适合继续审计的校准候选。

## 3. `m - 2tr(H) + tr(H^2)` 到底在修正什么

设观测有 `m` 个分量，正则化重建可以近似写成

```text
拟合观测 = H × 观测
```

这里的 `H` 可以理解为“帽子矩阵”或 influence matrix：它告诉我们第几个观测值有多少影响被拟合结果重新吸收。如果模型完全不拟合数据，`H` 接近 0；如果模型很灵活，`H` 的迹会变大，表示用了更多有效自由度。

普通残差平方和把 `观测 - 拟合观测` 当作噪声，但模型越灵活，残差越容易被压小。于是残差看起来可能比真实噪声小，并不代表重建真的更准确。

`m - 2tr(H) + tr(H^2)` 是残差对应的有效噪声维数修正。直觉上：

- `m` 是原本有多少个观测方向；
- `-2tr(H)` 扣掉模型已经用来追随数据的自由度影响；
- `+tr(H^2)` 加回正则化 influence 的平方项，避免只用一个粗略的 `m - 2tr(H)`；
- 合起来衡量“残差里还剩多少有效噪声方向”，因此比单纯把维数当成 `m` 更适合校准 discrepancy。

这不是把每个数据点真的删掉，也不是自动增加了独立样本数。它只是根据线性化的拟合算子，修正残差应该对应的噪声规模。若 `H` 的计算、协方差模型或线性化前提不对，修正后的数字仍然可能不可靠。

## 4. v5f：校准和重建分开了，但仍然不够

把校准和重建分开是正确方向。若用同一个 truth 同时决定正则化参数，又用同一个 truth 评价场误差，方法可能因为看到了答案而显得很聪明。v5f 明确记录了：校准和重建使用不同的 kappa；routing 也不使用 truth 或 audit camera。这样至少减少了直接的信息泄漏。

但“分开”不等于“独立验证已经完成”。v5f 的校准仍采用 diagonal DF-corrected discrepancy target 1.0，受限于当前的协方差近似；样本行也不是 IID，独立的 rig/session 最高层单位仍只有 2 个。若校准规则在一个 view、一个 session 或某种相机噪声结构下形成，换 view 或换 session 后，原来的半径可能失去意义。分离了两个 kappa，只能避免一类耦合，不能替代 view-specific transfer 和真正独立 session 的测试。

更直接的结果是：GCV 和 UPRE 最终给出了相同的重建结果。在 48 个样本中，两种方法都改变了 24 个半径，字段误差统计也完全相同：

- raw field mean error reduction：`-0.077%`；负号表示平均误差没有降低，反而略微增加；
- raw field p10：`-3.655%`，说明较差尾部没有改善；
- raw field harm rate over 1%：`25%`，四分之一样本的字段误差恶化超过 1%；
- outer gate：只有 `2/48` 个样本通过，覆盖率约 4.17%；
- 这 2 个被 gate 接受的样本，在 audit 上是 `2/2` 变差，audit harm rate 为 100%。

因此 v5f 的结论必须是 **NO-GO**：当前双正则路由不能进入冻结或论文中的成功叙述。它没有证明 GCV、UPRE 或双正则思想没有价值，只证明现有校准、转移和外层 gate 还不足以支持部署性结论。

## 5. 物理直觉：为什么要特别看 view 和 session

相机看到的同一个物理场，在不同 view 下可能经历不同的遮挡、投影角度、照明和响应；在不同 session 下还会有重新装夹、重新标定、温度、背景噪声和相机状态变化。一个在 view A 上看起来合适的正则化半径，不一定能直接转移到 view B。

如果训练和选择样本来自同一个 session，模型可能学到的是这次 session 的仪器状态，而不是稳定的物理规律。只有把完整 session 留在外层、在选择阶段完全不碰它，再检查另一个 view 或另一个独立 session 是否仍能工作，才是在问真正的外推问题。

所以后续最优先的不是再增加更多后处理规则，而是解决：

1. **view-specific transfer**：明确每个 view 的观测几何、噪声和标定差异，验证是否需要按 view 校准，或建立有物理依据的跨 view 转移规则。
2. **independent session**：使用真正独立的新 session 做外层评价；session 的划分要先冻结，不能看完结果后重新划分。
3. 在这两个问题解决前，任何“平均误差略好”“某个准则偶尔通过”的结果都只能作为开发线索。

## 6. 给师兄的 5 个问题

1. v5f 的校准仍用 diagonal covariance；在 view-specific transfer 前，应该按 view 分别估计协方差和 target，还是先建立一个跨 view 的物理归一化？
2. 目前只有 2 个独立 rig/session 最高层单位，真正的 independent session outer split 具体准备留出哪些 session？哪些 session 在选择和校准阶段必须完全不可见？
3. residual-DF target 1.0 达到 4/6、0 个半径边界，但 outer MSE 仍约 172。我们应把 MSE 大的原因优先归因于协方差、视角转移、场尺度，还是重建模型本身？下一项诊断是什么？
4. GCV 和 UPRE 在 v5f 最终完全相同，是否意味着当前搜索网格或校准半径把两者的差异抹平了？如何设计一个不看 truth 的检查来确认它们确实在做不同的选择？
5. 在 raw mean 为 `-0.077%`、p10 为 `-3.655%`、25% harm，且 outer gate 仅 2/48、audit 2/2 变差的情况下，NO-GO 的解除标准应预先写成哪些字段、尾部和 audit 条件？

## 7. 四个学习练习

1. **手算有效自由度**：取一个简单的 ridge 线性模型，写出 `H`，计算 `tr(H)`、`tr(H^2)` 和 `m - 2tr(H) + tr(H^2)`；比较强正则化与弱正则化时这三个量如何变化。
2. **画残差校准图**：在同一组模拟观测上分别用对角协方差和完整协方差白化，画 raw discrepancy 与 DF-corrected discrepancy，解释为什么数值改变不等于重建变好。
3. **做 session 留出表**：把每个样本标成 view、rig、session 三层，设计一次只按独立 session 留出的 train/selection/outer-test 表，并指出哪些信息不能进入 selection。
4. **复现 NO-GO gate**：根据 `method_summary.csv` 计算 mean、p10、harm rate、2/48 coverage 和 audit 2/2 harm，写一段不使用 truth 来决定 route、但使用外层结果来审计 route 的流程说明。

## 8. 本阶段结论

v5e 说明 exact covariance oracle 几乎没有改变方法排序，残差-DF 修正带来了更规整的边界行为，但没有消除较大的外层误差。v5f 说明校准与重建分离是必要的工程结构，却不足以克服 view/session 转移和非独立样本问题。双正则结果应保留为失败但有信息量的诊断，当前明确 **NO-GO**；没有创新成功、没有可冻结赢家，也没有可写成 superiority 的证据。
