# V5H-V5R 共享场逆算子研究日志

日期：2026-07-16

范围：`v5h`、`v5i`、`v5j`、`v5k`、`v5l`、`v5m`、`v5n`、`v5o`、`v5p`、`v5q`、`v5r`

证据边界：合成弱偏折机制开发；不是 OpenBOS/OERF 真实实验，也不是论文成功证据

## 1. 先给结论：这不是一条不断“刷高分”的成功故事

这一支线真正学到的是，评价逆问题算法时，**起点、对照、统计单元和打开数据的顺序，比一个正的平均数更重要**。

最初的 GC-RIO v0 曾报告约 `+0.035%`，但当时最强基线集合里没有 `zero_correction`。补上“完全不做残差修正”后，v0 实际是 **-4.083% / 0-of-4**，signed-probe v1 是 **-4.270% / 0-of-4**。这不是把旧结果偷偷改掉，而是纠正了比较对象：旧 `0.035%` 只表示 v0 比 analytic/no-target/shuffled 中的最强者略好，并不表示它比“不修正”好。

随后 oracle 诊断显示问题并非毫无可学空间：知道真值或干净目标残差时，理论 headroom 可达 **51.0% / 66.8% / 70.6%**。于是方法从“针对每个 target query 生成修正”改成“只根据 source 生成一个共享三维修正场，再由不同 target operators 解码”。

共享场 v2 的正式 Krylov 候选仍未通过：**-0.904% / 1-of-4**（相对每个单元的最强基线）。但是同轮未做预测集成的 adjoint-only 分支，相对 zero baseline 为 **+4.285% / 3-of-4**。看过这组结果后才做的三种子 ensemble，在旧 validation 上得到 **+6.329% / 4-of-4**，三维场平均改善 **+5.07%**、40 个场中 **70%** 改善。这些都是 post-open 诊断，不能倒写成预注册成功。

最后，不重训地把同一 ensemble 放到 4 个新 rigs × 2 个新 topology families 上，target 平均仍为 **+4.736%**，但只有 **3/8** 单元为正，最坏单元 **-8.287%**；三维场平均 **+3.692%**、128 个场中 **76.56%** 改善。由于 target 稳定性和尾部风险都不合格，最终仍是 **`NO_DESIGN_LOCK_OPEN`**。

更关键的是，v5n 加入训练集选定的 32 步 source-only PBB 后，纯 ensemble 相比 PBB 的总体 RMSE **差 15.83%**，逐 cell 平均差 **18.93%**，只有 **1/8** cell 胜出。于是“单次网络推理可作为独立重建算法”的叙事被淘汰。v5o 将网络降级为三维先验和 warm start，再做 prior-anchored PBB：训练集自动选择的 32 步版本只比同代理预算 PBB 好 **0.405%**，仅 **4/8** cells 改善、最坏退化 **9.44%**；但事后预算曲线在历史 proxy `B=11` 时出现 **+3.378%** 的低预算信号。它只用于生成下一次固定假设。

V5P 随后在首开前把代码与配置提交为 `35365ec`，再一次性运行 6 个新 rigs × 3 个新 topology families × 20 fields。精确计数下，候选每场 `8F/9A`，PBB-9 为 `9F/9A`。候选主增益只有 **+2.575%**，低于 3%；正 cells **8/18**，低于 75%；最坏 **-6.643%**，超过 5%。本机总时间约为 PBB-9 的 **5.14 倍**。三条预注册门槛全部失败，因此低预算加速假设也被判为 **`FRESH_DEVELOPMENT_NO_GO`**。

V5Q 在不改模型与判决的前提下做 post-open 机理审计。candidate 相对 PBB-9 的 source residual gain 与 target gain 在 field/cell 层 Spearman 为 **0.554 / 0.802**，6/6 rigs 同向；但自然零阈值 fallback 只带来 **+1.543%** 整体增益，正 cells **55.56%**，且被选场仍有 **22.77%** 受伤。因此它只是下一次 reserved-view reliability 实验的候选变量，不是已经成功的 gate。

V5R 随后使用每个 rig 原本封存、从未进入 reconstruction 的第 7 个 reserved view 做事后审计。reserved gain 与两台 target 的 field/cell Spearman 为 **0.548 / 0.674**；自然符号门把整体增益提到 **+3.405%**、最坏 cell 退化压到 **0.986%**，但只有 **12/18** cells 为正，被选场伤害率反而达到 **33.71%**，且 rig D 的相关仅 **0.016**。所以 reserved camera 改善平均数，却没有形成稳定安全门；可靠性符号规则在这批数据上正式归档为 NO-GO。

## 2. 共同物理问题和评价公式

把三维场离散为 \(f\)，source 与 target 相机的前向算子分别记为 \(A_s,A_t\)。source 基线场是 \(f_{base}\)，source 仍未解释的残差为

\[
r_s=y_s-A_s f_{base}.
\]

模型只根据 source 信息估计修正场 \(\Delta f\)，再用目标相机算子做显式物理解码：

\[
\hat r_t=A_t\Delta f,
\qquad
\hat f=f_{base}+\Delta f.
\]

推理时可以使用 \(A_t\)，因为它是相机几何和前向模型；不能使用目标观测 \(y_t\)，因为要预测的正是 target residual。训练标签是带噪目标残差，但标签只进入 loss，不进入 predictor。

每个 target row 的主要误差是报告所称的 whitened target residual RMSE：

\[
e_i=\sqrt{\frac{1}{M}\sum_{m=1}^{M}
\left(\frac{\hat r_{t,im}-r_{t,im}}{\hat\sigma_{t,i}}\right)^2}.
\]

候选相对基线的增益为

\[
g=1-\frac{e_{candidate}}{e_{baseline}}.
\]

每个 `rig × family` 先形成一个 cell，再对 cells 等权平均。数值越大越好；负数表示候选比基线更差。

## 3. V5H：zero baseline 纠正了旧 0.035%

证据：[v5h report](../demo_t16_operator/results/v5h_gc_rio_development/report.json)

GC-RIO v0 用 source 残差的伴随回投影、source Fisher 对角线和 target Fisher 对角线作为 3D CNN 特征，并在解析修正 `analytic_correction` 上增加一个受限的学习增量。它最初相对 `analytic_dc`、no-target 和 shuffled 对照中最强者得到约 `+0.035% / 2-of-4`。

后来加入 `zero_correction`：令

\[
\Delta f=0,\qquad \hat r_t=0.
\]

这代表“接受 source 拟合出的 \(f_{base}\)，不再声称残差可迁移”。它简单，却是残差校正方法不可缺少的基线。加入后，zero 在四个 validation cells 中全是最强基线，v0 的逐 cell 增益为：

| Cell | V0 相对 zero |
|---|---:|
| validation-rig-a × jet_shear | -2.883% |
| validation-rig-a × shock_cell | -4.020% |
| validation-rig-b × jet_shear | -5.431% |
| validation-rig-b × shock_cell | -3.996% |
| **Cell 等权平均** | **-4.083%** |

所以正确口径是 **-4.083% / 0-of-4 / `NO_DESIGN_LOCK_OPEN`**。旧 `+0.035%` 仍可作为“相对旧基线集合”的历史计算保留，但不能再写成 v0 的有效收益。

## 4. V5I：signed probes 没有修好错误起点

证据：[v5i report](../demo_t16_operator/results/v5i_signed_probe_development/report.json)

V1 加入 signed target row-space probes，希望补回 Fisher 对角线平方后丢失的方向与符号信息。这个想法针对的是 target geometry 表达能力，但它没有解决更根本的问题：网络仍围绕一个已经输给 zero baseline 的解析修正做有限幅度调整。

V1 的逐 cell 增益为 `-3.072%`、`-4.317%`、`-5.661%`、`-4.029%`，平均 **-4.270% / 0-of-4**，仍是 `NO_DESIGN_LOCK_OPEN`。这说明“给网络更丰富的 target query”不等于“修正场就会更正确”；如果基线修正的方向本身不可靠，表达能力增强也可能只是更复杂地拟合错误目标。

## 5. V5J：oracle headroom 说明有空间，但不是可部署结果

证据：[v5j report](../demo_t16_operator/results/v5j_gc_rio_headroom_diagnostic/report.json)

在 v5h/v5i 已经做出 no-open 决定后，v5j 才打开 development truth 做事后诊断。它问的不是“现有模型多好”，而是“这个任务在当前合成器里究竟有没有可利用空间”。以 `analytic_source_correction` 的 validation noisy error 为参照：

| Oracle | 它偷看了什么 | Headroom |
|---|---|---:|
| `true_field_correction_oracle` | 真值三维场差 | **51.0%** |
| `paired_target_field_oracle` | 同一场的两个干净 target residual | **66.8%** |
| `clean_measurement_oracle` | 干净 target residual | **70.6%** |

三者都不可部署。它们读取真值或干净目标信息，只能作为上限诊断。结果说明：

1. 当前生成器中确实存在可预测结构，不是所有 target residual 都是随机噪声。
2. 真值场修正只有 51.0%，低于 clean-measurement 的 70.6%，说明重建模型算子与 truth operator 的失配会让“真实场差”无法被当前 decoder 完整表达。
3. 两个 target views 联合拟合共享场可到 66.8%，支持尝试“一个三维修正解释多个视角”，但不证明 source-only 模型能达到这个上限。

这里必须保留时间顺序：oracle 是 **post-open development diagnostic**。它可以启发 v2 架构，不能被写成 v0/v1 之前就预注册的依据，更不能当成模型结果。

## 6. 为什么改成共享场，并把输出头初始化为 zero

实现：[shared_field_model.py](../demo_t16_operator/gc_rio/shared_field_model.py)

### 6.1 共享场

V0/V1 的 learned field 会读取 target-side geometry query，因此同一个物理场的两个 target views 可能得到两个不同的 \(\Delta f_t\)。这在数学上能拟合标签，但物理解释不够硬：同一个瞬间、同一个三维场，不应因为“准备给哪台相机看”而变成两个不同的三维修正。

V2 改为

\[
\Delta f=\mathcal N_\theta(A_s,r_s,\hat\sigma_s,f_{base},support),
\qquad
\hat r_{t,j}=A_{t,j}\Delta f.
\]

学习器不读取 target geometry；同一 source 条件只生成一个 target-independent correction field。不同 target cameras 只在最后通过各自 \(A_{t,j}\) 解码。这样，target supervision 约束的是同一个共享三维原因，而不是两个相机专属答案。

### 6.2 Zero initialization

共享场模型的最后一层权重和偏置严格初始化为 0，因此训练开始时

\[
\Delta f_{\theta=0}=0.
\]

代码中的 `proposed` 直接等于学习场乘 support，不再把 `analytic_correction` 加到输出上；当前 `data_consistency_step=0` 时，初始预测精确等于 zero correction。这样做有三层意义：

- 起点就是目前最强且最保守的“不修正”基线；
- 学习器必须从 loss 中赢得每一份非零修正，而不是继承一个已知有害的 analytic correction；
- 输出尺度改为最多占 `base_field` RMS 的 55%，不再被很小且可能方向错误的 analytic residual fit 限死。

Zero init 只保证初始函数安全、便于优化，不保证训练后仍优于 zero；最终仍必须用新数据和强基线检验。

## 7. V5K：正式 Krylov 候选失败，但 adjoint-only 分支出现信号

证据：[v5k report](../demo_t16_operator/results/v5k_shared_field_development/report.json)

V2 的 source Krylov 特征从 \(x_0=0\) 开始，做三步 Jacobi 预条件迭代：

\[
x_{k+1}=x_k+\omega
\frac{A_s^T\Sigma_s^{-2}(r_s-A_sx_k)}
{\operatorname{diag}(A_s^T\Sigma_s^{-2}A_s)+\lambda},
\qquad k=0,1,2.
\]

三个归一化迭代场作为 3D CNN 的 Krylov channels。`adjoint-only` 消融则把同一个预条件伴随场重复到三个 channels，不向网络提供逐步迭代轨迹。

### 7.1 官方候选口径

预先指定的正式候选是 `sf_rio_krylov_stack`。它必须在每个 cell 中与 zero、analytic、adjoint-only 和 shuffled 中的**最强**基线比较。结果是：

| 指标 | V2 Krylov 官方候选 |
|---|---:|
| 相对最强基线的 cell 平均增益 | **-0.904%** |
| 正 cell | **1-of-4** |
| 最坏 cell 退化 | **-2.261%** |
| 决定 | **`NO_DESIGN_LOCK_OPEN`** |

它相对 zero 自身其实是 `+3.973% / 3-of-4`，但这不能替换官方判断，因为更强的 adjoint-only 基线已经在部分 cells 出现。候选选择规则不能看到结果后临时改成只与较弱的 zero 比。

### 7.2 Adjoint-only 的事后观察

`sf_rio_adjoint_only` 相对 zero 的逐 cell 增益为 `+3.258%`、`-2.302%`、`+9.733%`、`+6.452%`，平均 **+4.285% / 3-of-4**。

这里的“单模型分支”是指每个 seed checkpoint 独立预测、独立评分，再对三个种子的 row errors 汇总；它不是先平均三个模型预测的 ensemble。因此这组数字可以说明“较简单的 adjoint-only 输入值得继续检查”，却不能改判 v5k，也不能把它倒写为正式候选。

## 8. V5L：post-open ensemble 变稳定了，但只能叫诊断

证据：[v5l report](../demo_t16_operator/results/v5l_shared_field_ensemble_diagnostic/report.json)

看过 v5k 后，v5l 取三个已存在的 adjoint-only checkpoints（seeds 3101/3102/3103），不加权平均它们预测的三维修正场：

\[
\Delta f_{ens}=\frac{1}{3}\sum_{q=1}^{3}\Delta f_{\theta_q},
\qquad
\hat r_t=A_t\Delta f_{ens}.
\]

在同一旧 validation 上，target cell 结果变为：

| Cell | Ensemble 相对 zero |
|---|---:|
| validation-rig-a × jet_shear | +5.151% |
| validation-rig-a × shock_cell | +0.368% |
| validation-rig-b × jet_shear | +11.605% |
| validation-rig-b × shock_cell | +8.192% |
| **Cell 等权平均** | **+6.329%** |

即 **4-of-4** 为正。打开 validation truth 后的三维场诊断为：平均 relative-L2 从 0.29430 降到 0.27542，平均 field gain **+5.07%**；40 个 validation fields 中 **70%** 改善。梯度 relative-L2 也从 0.30938 降到 0.28880。

但是 ensemble 规则是在看过 v5k 后才选择，truth-field 指标也在此时才加入。它们都是 `adaptive_postopen_development_diagnostic`。合理说法是“ensemble 在已反复使用的 validation 上呈现更稳定信号”，不能说“预注册 ensemble 通过”或“已经证明泛化”。

## 9. V5M：不重训的新 rigs/topologies 扩展仍然 no-open

证据：[v5m report](../demo_t16_operator/results/v5m_shared_field_extension/report.json) 与 [v5m 配置](../demo_t16_operator/configs/v5m_shared_field_extension.json)

V5M 保持三个 v5k adjoint-only checkpoint 的哈希不变，不重训、不在扩展标签上选 checkpoint，也先哈希预测再评分。扩展包含：

- 4 个新 rigs：`extension-rig-a/b/c/d`；
- 2 个新 topology families：`vortex_ring_pair`、`wrinkled_flame_sheet`；
- 每个 rig-family 16 个三维场，每场 2 个 target views；
- 共 128 个场、256 个 target rows、8 个 `rig × topology` cells。

结果汇总为：

| 指标 | V5M 不重训扩展 |
|---|---:|
| Target cell 平均增益 vs zero | **+4.736%** |
| 正 cells | **3/8（37.5%）** |
| 最坏 cell | **-8.287%** |
| 3D field 平均 gain | **+3.692%** |
| Field better fraction | **76.56%（98/128）** |
| 决定 | **`NO_DESIGN_LOCK_OPEN`** |

Target 的 8 个 cell gains 是 `-2.015%`、`-2.643%`、`-4.614%`、`+13.614%`、`+20.939%`、`+24.416%`、`-3.521%`、`-8.287%`。平均数为正，主要由 extension-rig-c 的两个大正值拉高；超过一半 cells 为负，而且尾部伤害越过预设 5% 上限。

门禁要求平均至少 10%、至少 75% cells 为正、最坏退化不超过 5%。V5M 三条都没有通过，所以不能因 field 指标的 `+3.692% / 76.56%` 而打开 design lock。Target residual 与 3D truth field 是两个不同问题：场的全局 L2 变好，不保证每个新相机几何下的投影残差稳定变好。

V5M 比反复看旧 validation 更有价值，因为 checkpoint 未重训且数据是新构造的；但这些 rigs/topologies 是在看过 v5k/v5l 后设计的，仍属于 adaptive development extension，不是原始 design lock，也不是确认性真实数据。

## 10. V5N：强 PBB 基线否掉“纯网络重建”

证据：[v5n report](../demo_t16_operator/results/v5n_strong_classical_baselines/report.json)、[逐 cell 表](../demo_t16_operator/results/v5n_strong_classical_baselines/extension_cells.csv)

V5N 没有在 extension 标签上调参。它先在原 train split 上一次性选择：

- support-ridge 相对正则系数为 `1.0`；
- source-only projected BB 的迭代数为 `32`；
- 两者只读取 source residual、source operator、support、base field 与每相机标量 sigma；
- 冻结后才在 v5m 的 4 个 rigs × 2 个 topology cells 上评分。

结果是：

| 方法 | Extension cluster-mean standardized RMSE | 3D field mean relative-L2 |
|---|---:|---:|
| Zero correction | 6.0288 | 0.23455 |
| Support ridge | 6.0311 | 0.23472 |
| Shared-field ensemble | 5.6119 | 0.21884 |
| **Projected BB-32** | **4.8450** | **0.16828** |

因此 ensemble 相对 PBB-32 的总体 ratio-of-cluster-means gain 为 **-15.829%**；逐 cell 等权平均为 **-18.928%**，仅 **1/8** cell 为正，最坏退化 **50.69%**。三维 truth diagnostic 也一致：PBB 相对 base 平均改善 **24.98%**、`96.09%` fields 变好；ensemble 只有 **3.69%**、`76.56%` fields 变好。

这一步改变了研究定位：共享场网络不再被当作“单独击败数值逆解器的通用重建器”。它最多可能是一个 amortized prior、warm start 或低调用预算加速器。PBB-32 与 CNN 的 wall-clock 尚未匹配，所以 V5N 不能证明 PBB 在所有成本口径都优越；但它已经足以否定“只和 zero/analytic 比就能宣称算法成功”。

## 11. V5O：共享场先验加 PAPBB 的低预算假设

实现：[v5o runner](../demo_t16_operator/run_v5o_prior_anchored_frontier.py)；证据：[v5o report](../demo_t16_operator/results/v5o_prior_anchored_frontier/report.json)、[预算前沿](../demo_t16_operator/results/v5o_prior_anchored_frontier/extension_frontier.csv)、[证据图](../demo_t16_operator/results/v5o_prior_anchored_frontier/v5o_frontier.png)

V5O 把网络输出记为 \(\Delta f_\theta\)，不再直接作为最终答案，而是求解一个带学习先验锚点的 source-only 目标：

\[
J(\Delta f)=
\left\|\hat C_s^{-1/2}(A_s\Delta f-r_s)\right\|_2^2
+\alpha\left\|\Delta f-\Delta f_\theta\right\|_2^2,
\]

并在每一步后投影到 support 与非负 corrected-field 约束。直观地说，网络负责提出“PBB 从哪里开始、哪些源算子不可见方向可能值得保留”，PBB 负责让结果重新服从真实 source residual。

锚点系数与迭代数仍只在 train split 选择。训练集自动选出 `relative_anchor=0.1`、`32` 次 refinement；V5O 当时沿用 `3+k` 作为历史 operator-pair 代理预算。后续代码审计发现 adjoint-only checkpoints 可以在三个 seeds 间复用一份 source adjoint/Fisher，而且不必执行随后被丢弃的 Krylov stack；这一优化保持预测哈希不变。V5P 因此不再沿用 `3+k`，而改为逐次记录真实 `A_s/A_s^T` 调用。V5O 冻结到 extension 后：

| 方法 | Proxy budget | Cluster-mean RMSE | 相对预算匹配 PBB |
|---|---:|---:|---:|
| Pure PBB | 35 | 4.82175 | baseline |
| **SFIO-PAPBB（train-selected）** | **35** | **4.80224** | **+0.405%** |

这个均值没有通过稳定性门禁：逐 cell 平均是 **-0.651%**，仅 **4/8** cells 为正，最坏退化 **9.438%**。因此正式结果仍为 `NO_DESIGN_LOCK_OPEN`。

## 15. 红队结论：目前证据还有五层结构性风险

### 15.1 同一 validation 被反复使用

V5H/V5I 用 validation 选 early-stopping checkpoint 并做门禁；V5J 打开同一 development truth；V5K 虽在 train split 上拟合参数，却又用同一 validation 选择 checkpoint、比较新架构；V5L 看完 V5K 后在同一批数据上选择 ensemble 与 field 指标。每一次查看都会把信息反馈进下一步设计。即使没有直接把 validation rows 加入训练梯度，整条研究过程也已经对这批 validation 发生人为过拟合。

所以 V5L 的 `+6.329% / 4-of-4` 只能是 post-open 诊断。V5M 虽然是新数据且不重训，但其数据设计也发生在前述结果之后，只能提供一次更诚实的扩展压力测试。

### 15.2 Rows 不是独立样本

同一个三维场生成两个 target rows；同一个 rig-family 下 16 或 20 个场共享 operator、噪声机制和几何；V5K 还对同一 rows 重复评估三个 seeds。不能把 80、240 或 256 rows 当作同样数量的独立物理实验。

最低限度应以 `field_uid` 为 cluster，把两个 target views 和多个 seeds 留在同一 cluster 内；再按 `rig × topology/family` 报告 cell 分布。若做置信区间，应按 field cluster bootstrap，并把 rig 当更高层级，而不是对 rows 做普通 IID t-test。

### 15.3 Scalar sigma 不是真正 whitening

当前 `paired_difference_mad` 为每台相机估计一个标量 \(\hat\sigma_c\)，等价于假设

\[
\Sigma_c=\hat\sigma_c^2 I.
\]

将所有 pixels 除以同一个数只是 camera-wise standardization。真实相机常有空间异方差、固定图样噪声、相邻像素相关、时序漂移和跨相机公共扰动；此时真正 whitening 应使用 \(C^{-1/2}r\)，满足白化后协方差接近单位阵。当前指标名称沿用 `whitened_rmse`，但技术上只能说“按每相机 scalar sigma 标准化的 RMSE”。

下一次真实数据实验至少要用重复 flow-off/reference frames 估计 pixelwise diagonal variance；若重复数足够，再检查低秩公共模态或 block covariance，并报告白化后残差的空间相关和相机间相关是否仍存在。

### 15.4 Shuffled control 不够干净

V0/V1 的 shuffled target control 调换 conditioning query，却仍用真实 target operator 解码；V2 的 shuffled source control 调换 conditioning source operator，但 source residual、真实 source operator和最终 target decoder并没有形成一套完全一致的反事实样本。这样的错配可能把输入推到训练分布之外，同时改变几何语义、数值尺度和 operator-residual 一致性。

因此，`correct - shuffled` 的差不能直接解释为“模型使用了几何信息”的纯因果效应。更干净的 control 应保持 operator 的谱、噪声尺度、rig 分布和 residual 一致性，只破坏待检验的配对关系；或者在独立生成器中重新生成物理一致的反事实 paired samples。

### 15.5 与 CG-PDNO/PBB 的方法边界重合

共享场模型使用 source adjoint、Fisher/Jacobi 预条件、三步迭代轨迹、学习修正、support projection 和显式 physics decoder。这与仓库已有 CG-PDNO、regularized CG、projected Barzilai-Borwein（PBB）轨道在核心结构上明显重合。

因此现在不能把“shared-field inverse operator”直接包装成全新算法。更保守的定位是：**一个用 cross-view supervision 训练的 source-only learned correction head，建立在经典迭代逆问题特征之上。** 是否有额外价值，必须在相同 source inputs、相同正则、相同 support、相同 forward/adjoint 调用预算下，正面对比 PBB 和 regularized CG。

## 16. 下一步唯一合理工作

V5P 已按约定失败，因此现在不能再给同一网络追加 attention、更多 seeds、family router 或重新选择 anchor/k，然后把它叫作“改进版”。V5Q/V5R 已完成 source 与 reserved-view 失败机理审计；两个自然门都没有把 conditional harm 和 rig 稳定性降到可接受水平，当前主线执行路线 B。

### 16.1 已完成的可靠性诊断

V5Q 已在 V5P 打开数据上计算以下 source-observable quantities，但没有据此改判：

- prior 与 PBB-9 的场差、夹角和 source-projected difference；
- candidate/PBB 的 source objective、非负投影激活比例与 support 边界比例；
- 三 seed prior disagreement、adjoint/Fisher 可见性和近零空间能量 proxy；
- 上述量与 target gain 的 field-cluster 相关，以及在三个 topology families 间的分布。

结果说明 source/reserved residual 的逐场排序关系可以作为未来真实数据的辅助诊断，但自然门分别伤害 22.77% / 33.71% 被选场。任何新阈值、分类器或相关系数都属于 post-open hypothesis generation，不能在 V5P 上报告“路由后提升”。

### 16.2 主线转入路线 B

**路线 A：暂时关闭。** V5R 已使用未参与重建的 reserved camera；平均数改善但正 cells 只有 66.67%、selected harm 33.71%，且 rig D 几乎无排序关系。除非真实 OERF 数据提供独立 session 和明确的 audit-camera 使用条件，否则不再给旧 fields 追加 gate。

**路线 B：停止学习 prior。** 如果 source-observable quantities 无法区分帮助与伤害，就把当前 SFIO-PAPBB 归档为完整负结果，把主线转向更贴近 OERF 的 forward mismatch、flow-off covariance、cone/curved-ray correction，或 TDBOST 的时空正则与稀疏创新。此时 PBB/CG 是重建 backbone，学习只处理有独立证据的模型误差或时间先验。

无论选择哪条路线，regularized CG、PBB-9/11/32、zero 和纯 prior 都继续作为基线；逐次 `A/A^T`、CNN wall-clock、内存、field cluster 和最坏 rig-family 都必须保留。V5P 已经证明“算子调用少一点”不等于本机总时间更快。

### 16.3 Field-cluster 统计

统计表应一场一行，而不是一 target view 或一种子一行：

- 主 cluster：`field_uid`；同场的所有 target views、seeds 和重复测量不得拆开；
- 上层分组：rig、topology/family、session；
- 主指标：held-out target standardized residual，以及有可信 truth 时的 3D field relative-L2；
- 次指标：gradient relative-L2、最坏 rig-family degradation、positive-cell fraction；
- 区间：按 field cluster bootstrap，并单列每个 rig/topology，不用大量相关 rows 缩小置信区间。

### 16.4 真实 OpenBOS/OERF 数据合同

仓库已有 [真实数据接口模板](../README.md#当前真实数据接口模板) 与 [measurement_contract.py](../demo_t16_operator/measurement_contract.py)。下一轮在读取任何 target labels 前，应冻结一个实际可填充的数据合同：

| 类别 | 必需字段或文件 |
|---|---|
| 身份与权限 | dataset/session/rig/field/view ID，来源、license、可公开范围、原始文件哈希 |
| 原始测量 | REF/DEF 图像对或 displacement field，单位、算法版本、confidence/outlier mask |
| 几何 | 相机内外参、view angle、rotation-to-angle 映射、坐标系、标定版本及不确定度 |
| 有效区域 | ROI、support、坏点/遮挡 mask、每视角覆盖率 |
| 噪声 | 独立 flow-off/reference repeats、时间戳、曝光、帧率、pixelwise variance/covariance 估计 |
| 前向模型 | 可复现的 `forward`/`adjoint`，ray/aperture 假设、离散网格和算子哈希 |
| Split | source/target/reserved views，train/development/fresh-test sessions，field-cluster ID |
| 外部真值 | 若存在，记录 CFD/phantom/PLIF/其他 truth 的配准、分辨率和不确定度；不存在就不得制造 field-L2 |

OpenBOS 当前模板已列出 70 个 REF/DEF view pairs，但 ROT-to-Angle 映射仍需从论文或脚本确认，不能从目录名猜。OERF 组内数据还应先回答权限、相机标定、位移提取、reference repeats 和可公开字段。合同不完整时，可以做数据健康报告，不能先跑模型再补 metadata。

6. V5P 中低预算 hybrid 三条门槛全部失败，而且本机比 PBB-9 慢约 5.14 倍。您是否同意停止给当前 prior 追加结构，先转向 forward mismatch / covariance / 4D temporal prior？哪一类真实组内误差最值得优先处理？
7. OpenBOS 的 ROT-to-Angle 映射、REF/DEF 配对、mask 和 displacement confidence，您是否有可信脚本或论文说明可核对？哪些字段绝不能从目录结构推断？
8. V5R 已实际模拟固定一台 audit camera：平均增益提高，但只有 12/18 cells 为正、selected harm 33.71%，一套 rig 相关仅 0.016。真实九路装置是否有独立 session/重复视角足以学习经过校准的风险，而不是符号门？若没有，我计划停止可靠性 gate，主攻 forward mismatch 或 4D 时空先验，您更支持哪一个？

## 19. 最终边界

这条支线目前支持的最强陈述是：**共享三维修正场比 target-specific correction 更符合一个物理场解释多视角的结构；但纯 shared-field ensemble 明确输给强 PBB，预注册的低预算 prior-anchored hybrid 也没有在全新形态上达到稳定性与速度门槛。学习 prior 的收益表现出强 topology dependence，当前不能称为通用算子或通用加速器。**

不能声称：V0 的旧 `+0.035%` 是有效提升、Krylov v2 通过、post-open ensemble 是预注册结果、三维 field 改善等于 target 泛化、V5M/V5P 是 design lock、V5O 已证明 `+3.38%`、V5P 的 `+2.575%` 是 gate pass、少一次 forward 等于实际加速、scalar sigma 已完成真正 whitening、shared-field 是独立于 CG-PDNO/PBB 的新算法，或任何 OpenBOS/OERF 真实实验与论文成功。

当前决定仍是 **`NO_DESIGN_LOCK_OPEN`**，且当前 SFIO-PAPBB 主候选与 residual-sign reliability 支线都暂停。V5Q/V5R 表明 source 与 reserved consistency 能提供排序，但不能形成稳定安全门。下一步执行路线 B：PBB/CG 保留为 backbone，学习模块只处理有独立物理证据的 forward mismatch、噪声协方差或 TDBOST 时空先验；真实 OpenBOS/OERF 数据合同必须先于任何新 superiority 检验冻结。

## 20. 路线 B 的首次落地：V5S-V6A 结构漏斗

路线 B 随后完成了 9 个可复现实验。完整的讲人话解释见 [3D 逆问题学习持续日志](operator_3d_learning_log.md)，冻结合同与下一门禁见 [DCO/TRAIL 研究合同](route_b_dco_trail_research_contract_2026-07-16.md)。这里保留决定链：

1. **V5S 全局 GC-BiLOC 失败。** 共享 24×24 低秩修正 error 0.8285，明显差于 full-matrix geometry ridge 0.5217；前 16 个 measurement/voxel 奇异方向只解释 34.0%/38.4% 能量。
2. **V5T-V5V 连续排除错误表示。** truth-parameter 一阶 tangent 不够；truth geometry 对齐只移除 8.39% discrepancy norm；每相机固定 5×5 kernel 的 oracle 仍为 0.9043。
3. **V5W 隔离有限孔径。** 全局固定 kernel 只比 full-matrix ridge 改善 0.854%，说明孔径效应不是整幅图共享的 blur。
4. **V5X 找到逐射线局部结构。** 每 ray 3×3×3 voxel-kernel oracle 为 0.3587，但先拟合 kernel 再回归的两阶段方法失败；中间 kernel 不唯一。
5. **V5Y/V5Z 区分优化失败与结构信号。** 原优化器发散到 1.9769；稳定化的 891 参数线性模型达到 0.7707，相对 full-matrix ridge 改善 5.359%，worst-rig ratio 0.777。
6. **V6A 小型 hypernetwork 达到 8.080% near-signal。** 三种子 ensemble 为 0.7485，aggregate worst ratio 0.761，但只有 6/12 rigs 胜出，逐 rig 最大退化仍为 13.69%；未达到 10%/75% 开发门槛，更不满足未来 5% 安全门。

当前工作假设收窄为 **RayKernel-DCO**：根据相机内每条 ray 的几何生成位置相关局部 3D kernel，并从同一 kernel 严格构造 forward/adjoint。它只是 opened synthetic operator-level candidate；没有真实 OERF、fresh gate、limited-query calibration 或 inverse superiority。V6A 判决明确要求 **停止在当前数据上继续扩容**，下一有效动作是向何远哲确认真实有限孔径/景深失配，或者改用带 timestamp 的 TDBOST 连续序列启动 TRAIL-4D。

## 21. 外部证据与 V6B 少查询协议：现在到底缺哪一块

完整审计见 [公开 BOST 外部基准审计](public_external_bost_benchmark_audit_2026-07-16.md)，冻结规则见 [V6B-1.0 少查询预注册协议](v6b_limited_query_preregistration_2026-07-16.md)。这里把两份文件合成一条讲人话的决定链。

### 21.1 PSU 是公开真实数据，但它只是一套固定 rig

[Penn State 70-view flight-body BOS 数据](https://doi.org/10.26208/1VE2-5C19)提供 7 台相机、10 个模型转角、标定、flow-off/on、mask、三种 deflection 结果以及 NIRT 代码和重建产物。官方流程从相机 2--4 的 0°、50°、90° 组合出 9 views，因此它很适合做三件实事：检查 loader、单位、坐标手性和 mask；复现固定装置上的 NIRT；冻结模型后用其余视角检查重投影误差。

但“70 个公开视角”不等于“公开的跨光圈算子数据”。数据中的 f/32 与 f/22 来自不同相机/镜头，并不是同一物理场、同一相机只改变 f-number 的受控 sweep。它也没有独立三维实验真值。因此 PSU 能提供的是 **真实固定 rig 的 reconstruction/reprojection 证据**，不能单独证明 RayKernel-DCO 会跨 aperture 泛化，也不能在没有 truth 时报告 field relative-L2。

许可边界也要单列。官方元数据写 `Access_Constraints: None`，但 `Use_Constraints` 不是标准 CC/OSI 许可证，DataCite `rightsList` 为空，并对定制或增值工作保留通知要求。现阶段只在本机私有目录转换和复跑；公开仓库只放 DOI、我们自己的 loader 接口、指标和审计，不重发原 ZIP、作者源码、checkpoint 或派生数据。论文若实际使用并改造该包，投稿前还要向 Penn State 数据联系人确认通知方式。

### 21.2 `photon` 能制造新光圈和新布局，但受 CUDA 与合成证据约束

作者仓库 [`lalitkrajendran/photon`](https://github.com/lalitkrajendran/photon)是 PIV/BOS synthetic image generator，根目录明确给出 GPL-3.0，并要求编译 CUDA ray-tracing shared library；其公开说明和代码路径支持配置 f-number、焦距、物距、相机姿态、NRRD 密度场并输出 rays。因而它适合作为与本仓库 generator 独立的 fresh-A/B/C renderer，在 NVIDIA/CUDA 服务器上生成新的 aperture、layout 和 joint-OOD 测试。

边界同样明确：当前 Mac 不能被当作它的原生可执行环境；GPL-3.0 允许在许可证条件下使用、修改和再分发代码，但不替第三方依赖或生成数据自动授予额外权利；仓库也没有现成、经核验的精确 `F^T`/VJP。故 `photon` 通过只能写“跨 renderer 的合成迁移成立”，不能写“真实光学成立”，更不能直接写“OERF 装置有效”。

### 21.3 cone-ray 论文证明问题真实存在，不等于 benchmark 已开放

[有限孔径 cone-ray BOS 论文](https://arxiv.org/abs/2402.15954)在模拟与真实实验中比较了 thin-ray 与 cone-ray，并覆盖 f/22 到 f/4；这是一条很强的物理证据，说明有限孔径和景深确实可能改变 forward model，且修正模型能改善重建。它支持我们研究 forward mismatch 的动机。

不过，开放论文阅读许可不等于数据和代码再利用许可。当前没有对应公开 benchmark、可调用高保真 oracle 或作者伴随实现，所以它的证据等级是“同行评审/作者论文支持物理机制”，不是“本仓库已独立复现”。PSU、`photon` 与 cone-ray 三者拼起来，仍缺同一协议下的独立 aperture/layout、受限 `F x` 查询、可核验 `F^T z` 和 inverse truth。

### 21.4 为什么还要向 NeRIF/TDBOST 请求数据与算子

[NeRIF 论文](https://arxiv.org/abs/2409.14722)给出九视角数值与火焰实验，是最接近 OERF 的静态三维协议，但公开页面只有论文及论文源文件入口，论文的数据可用性口径是合理请求，没有可直接冻结的完整训练包。[TDBOST 作者仓库](https://github.com/Hyz617/TDBOST)公开了代码树和部分 Drive 入口，适合审计张量分解、数据 shape 和训练流程；但根目录没有明确许可证，完整连续序列、投影器和可核验伴随也未齐。因此“仓库可见”只能支持阅读和接口审计，不能推定允许修改、重发或把缺失文件补成完整复现实验。

这正是完整 V6B 仍标为 **`UNCONSTRUCTED`** 的原因：目前没有一个资源同时具备独立 aperture、独立 layout、二者联合 OOD、限次高保真 `F x`、严格 `F^T z`/VJP、封存 fresh labels 和可信 inverse truth。V6A 的完整矩阵训练缓存也不能继承，因为它已经看过全部 operator rows 与 truth geometry；若把它带进 V6B，就不再是少查询适配。

### 21.5 27 gates 不是 27 次高保真查询

候选 hypernetwork 对每条 ray 输出一个 radius-1 的 `3 x 3 x 3 = 27` 通道局部核。进入新 rig 后，不重训整个 8,091 参数网络，只根据校准残差拟合 27 个标量 gates `g_q`：

\[
r_k=A_\star x_k-A_0x_k,
\qquad
g=\arg\min_g\sum_k\left\|\sum_{q=1}^{27}g_q C_{:q}\odot A_0S_qx_k-r_k\right\|_W^2
+\lambda_g\|g-\mathbf 1\|_2^2.
\]

这里 `27 gates` 是模型自由度，不是 oracle 预算。主协议仍是 `K=32` 次 **vector-query**：一次 `A_star x_k` 返回全视角测量向量，记 1 次 query，同时另报 `K x views`、ray 数、path/aperture samples 与 wall-clock。候选在 fresh rig 只能用这 32 个 calibration probes，不能调用高保真 adjoint；第 33 次查询必须报错。30 个 train rigs 与 12 个 development rigs可打开，fresh-A/B/C 各 16 rigs 必须封存；每个 fresh rig 的 32 个 operator probes 和 12 个 inverse fields 只能在模型与 prediction hash 固定后由独立 evaluator 解封。

同一局部核必须同时构造 forward 与 transpose：

\[
A_{corr}x=A_0x+\sum_q g_q C_{:q}\odot A_0S_qx,
\qquad
A_{corr}^Tz=A_0^Tz+\sum_q S_q^TA_0^T((g_qC_{:q})\odot z).
\]

这样 27 gates 才是可审计的少查询校准，而不是在 fresh rig 上偷偷继续训练。比较对象也必须吃完全相同的 `X,R,g` cache，包括 minimum-norm secant、逐 ray 27 系数 ridge 加图拉普拉斯，以及 geometry polynomial/B-spline ridge；不能在 fresh 上逐 rig 挑最好基线。

### 21.6 梯度要对真实残差求，不能只比较回投影方向

旧指标比较 `Ahat^T(Astar x)` 与 `Astar^T(Astar x)`，漏掉了候选预测和观测之间真正的残差，可能在 forward 已经错误时仍给出好看的方向一致性。若目标函数是

\[
\mathcal L(x)=\tfrac12\|Ax-y\|_2^2,
\]

正确梯度只能是

\[
\nabla_x\mathcal L(x)=A^T(Ax-y).
\]

因此要在零场、受扰动 truth、nominal/CGLS 中间迭代点分别构造各自的 `Ax-y`，再比较候选与高保真残差梯度；先通过 dot-product test 和 finite-difference check，才允许报告 cosine 与 relative gradient error。协议门槛是 cosine 中位数至少 0.99、5% 分位至少 0.95、不得出现负 cosine，且相对梯度误差中位数不超过 0.25。G2 operator 门不过就不解封 inverse，不能用后面的重建偶然性替前面的算子错误开脱。

## 22. V6B 协议自检：先证明不会偷看，再证明它会失败

2026-07-16 新增的不是 fresh V6B 结果，而是一套 **toy protocol conformance**。代码入口为 [`limited_query_calibration.py`](../demo_t16_operator/limited_query_calibration.py) 和 [`run_v6b_protocol_conformance.py`](../demo_t16_operator/run_v6b_protocol_conformance.py)，逐 rig 结果、首评分前哈希与图保存在 [`results/v6b_protocol_conformance/`](../demo_t16_operator/results/v6b_protocol_conformance/)。

### 22.1 防火墙现在能做什么

- `BudgetedForwardOracle` 只暴露 `measure(x)`；每次尝试都计数，第 `K+1` 次立即报错；
- fresh calibrator 不需要完整 `A_star` rows、truth geometry 或 truth adjoint；
- 已知 nominal `A0/A0^T` 与 27 个 ray-conditioned kernel channels 共同构造 matrix-free forward；
- adjoint 由同一组 shift/channel 权重严格转置，不另训一个网络；
- K 个观测只拟合 27 个 gates，模型和 prediction arrays 在 hidden scoring 前写入 SHA-256；
- 20 对 float64 dot-product 的最大 defect 为 `2.5895e-13`，并通过真实残差 `A^T(Ax-y)` 的中心有限差分测试。

这些检查说明代码接口符合少查询协议；它们不说明候选接近真实 BOS 算子。

### 22.2 为什么特意放一个模型外失败

自检含 6 个 in-class rigs 和 6 个 misspecified rigs，统一使用 `K=4/8/16/32`、32 个 hidden probes 与同查询预算的 minimum-norm secant。输入是 `4 x 4 x 4 = 64` 维，因此主预算 K=32 仍然欠定。

| K=32 中位数 | nominal | 同预算 secant | 27-gate |
|---|---:|---:|---:|
| in-class hidden-action relative L2 | 0.09869 | 0.06982 | `2.44e-12` |
| misspecified hidden-action relative L2 | 0.15537 | **0.10801** | 0.11876 |

左侧近零是“真值本来就由 27-gate 家族生成”的单元级正控制。右侧额外加入 12% Frobenius-norm 的模型外低秩残差后，27-gate 只从 0.155 降到 0.119，并在 K=32 被 secant 的 0.108 反超。这条负结果证明：查询防火墙不会自动把一个错误结构变成成功；未来 PSU、`photon` 或 OERF 中若出现类似残差，应回到结构诊断，而不是继续加网络容量。

首次草案曾用 27 维输入配 K=32，导致 secant 获得满秩输入基并精确识别 toy 全矩阵。该设置已被删除，配置改为 64 维，并由测试强制 `max(K) < input_size`。这是协议修正，不得把删除前的数值用于比较。

### 22.3 当前判决

报告判决是 `PASS_PROTOCOL_CONFORMANCE_ONLY`，且明确写入：

- `fresh_v6b_status = UNCONSTRUCTED`；
- `scientific_claims_unlocked = []`；
- in-class truth 与候选共享 toy nominal operator；
- out-of-class residual 仍是人工低秩项；
- 不允许解锁 inverse、external renderer、real BOS 或 OERF claim。

下一步代码工作不是继续美化 toy 曲线，而是：完成 V6A checkpoint/center/scale/offset 的冻结 loader、把 evaluator 分离到独立进程、补 B2/B3 同预算基线和 rig-cluster statistics，然后接 PSU 固定 rig loader；只有拿到独立 aperture/layout 或 OERF calibration contract 后，才创建真正的 V6B fresh runner。

## 23. 从 gate-only 失败到 DF-gated SRCO：一个新候选怎样被逼出来

V6B 的 misspecified toy 说明 27-gate 无法表示所有 operator residual。第一次补丁 V6C 把 gate 未解释的 K-query 残差做低秩校准：模型外 error 从 0.11999 降到 0.08383，却把 in-class calibration noise 从 0.000163 放大到 0.001970，因此明确判 `POST_OPEN_HYBRID_NOT_DOMINANT`。

第二次 V6D 不再永久开启补丁，而用 gate residual 相对独立噪声地板的正部信号比例控制幅度。全新 16+16 toy rigs 的 K=32 结果中，in-class 激活中位数为 0，误差保持 0.000178；模型外激活中位数 0.9997，误差降到 0.08768，并在 16/16 rigs 优于 gate。它仍是看过 V6C 后的二阶段 post-open 诊断，不能改判 V6B 或解锁论文主张。

完整公式、逐 rig 安全、Broyden/learned correction 碰撞与下一次预注册见 [SRCO / DF-gated SRCO 假设合同](srco_postopen_hypothesis_2026-07-16.md)。最重要的认识是：加性低秩更新不是新意；可能的新意只在 BOST 结构保持、flow-off 噪声触发、主动 probe、严格查询账本和一致 inverse 证据链。

## 24. 红队修正：ridge 残差噪声不能只写 `n-tr(H)`

复核 V6D 时发现，第一版用 `MK-tr(H)` 近似 gate 拟合后的噪声自由度。这个写法对未正则的正交投影成立，但一般 ridge 的 hat matrix 不再幂等，严格的同方差残差噪声自由度应为

\[
\nu_{res}=\operatorname{tr}[(I-H)^T(I-H)]
=MK-2\operatorname{tr}(H)+\operatorname{tr}(H^TH).
\]

实现现已按 `G^T G` 的特征值精确计算该 trace，并新增两类测试：一类与显式 hat matrix 逐项比较，另一类检查无正则时退化为 `n-rank(G)`。这一阶段仍按同方差噪声处理，重跑后的中间版本 K=32 in-class / out-of-class DF-SRCO 中位数约为 `0.00017827` / `0.08767794`，float64 最大 dot defect 为 `1.44e-12`；第 25 节的异方差修正随后把最终 defect 更新为 `2.88e-13`。这次稳定不等于真实稳健，只说明修正没有靠原近似制造结果。

还新增了 [DF-gated SRCO 方法论文工作稿](df_srco_manuscript_working_draft_2026-07-16.md)。它包含秩界、最小 Frobenius 范数插值、未查询正交补零作用和解析伴随四个可证明命题，也完整保留 V6C 噪声过拟合负例。工作稿明确不是预注册、不是投稿稿，也没有 DeepONet/FNO 公平比较；下一次有效证据必须来自新的 external renderer、PSU 固定 rig 或 OERF 独立 session。

## 25. 第二次红队修正：V6D toy 本身是异方差噪声

独立代码审计继续发现：V6D 按每个 probe 的 clean-output norm 设置噪声标准差，因此不同 probe 列的方差不同；上一版虽然把 ridge 的 `n-tr(H)` 修正成精确同方差 trace，却仍用总噪声能量乘自由度比例。这个近似和 toy generator 不完全一致，不能一边强调真实 BOS 的异方差，一边在自己的 toy 中忽略它。

现已让噪声生成器返回与 gate design 堆叠顺序一致的逐行对角方差，并直接计算

\[
E_{noise,res}=\operatorname{tr}[(I-H)\Sigma_{diag}(I-H)^T].
\]

新增测试把该快速公式与显式 hat matrix + diagonal covariance 逐项比较，并锁定 probe-block 方差顺序。V6D schema 升级为 `1.1-heteroscedastic-covariance` 后重跑：K=32 的 in-class / misspecified DF-SRCO 中位数仍为 `0.0001782702` / `0.0876779408`，misspecified 相对 gate 中位改善仍为 `28.7833%`；float64 最大 dot defect 为 `2.88e-13`。数字几乎不变只能说明旧近似没有制造这组 toy 现象，不能证明 DF gate 在真实相关噪声下有效。真实实验仍需 flow-off repeats、whitening 或完整 `tr(R Sigma R^T)` 审计。
