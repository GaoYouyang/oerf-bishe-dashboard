# v5c 研究学习日志（2026-07-16）

> 本轮正式结论：**NO-GO。**失败的是当前“联合选择孔径 `radius` 与正则强度
> `kappa` 的跨视角规则”，不是有限孔径候选算子的物理语义。严格门最后接受
> `0/48` 行，因此当前方法没有展示出可用覆盖率，也没有形成 accepted-only 风险证据。
>
> 证据边界：这是一个 `8×8×5`、线性弱偏折、规定好的合成 surrogate 上的
> first-open。它不是 confirmatory lock，不是真实 BOS，也没有独立审计进程。

## 1. 这一轮为什么要做

v5b M1 已经把前向模型的基本尺子校正到可用状态，但 learned selection 在 held-out
audit camera 上仍有明显尾部失败。v5c 因此不继续加网络，而是问一个更窄、更可证伪的问题：

> 只用 inner cameras 联合选择孔径和无量纲 ridge `kappa`，再经过“删样本的逐相机门”和
> 双向 outer-camera transfer gate，能不能既恢复孔径，又减少 v5b M1 的 audit-camera 伤害？

本轮还换成相对 v5b M1 新的 `jet_shear`、`shock_cell` 生成 family，并使用新的 seeds 和
noise levels。但必须明确：**这两个 family 参加了本轮 inner selection，所以它们不是相对
当前选择过程的 family-OOD；本轮真正 held out 的只有 cameras。**

## 2. 代码和统计上怎样防止“偷看答案”

这轮相对更早版本的主要改进，不是把结果做得更好，而是让“选择时看过什么”更可审计。

1. **先 inner、后 outer。** `radius` 和 `kappa` 只由 inner views 选择；outer views 在
   inner 决策完成后才用于门控和迁移评估，并按两台相机分别计分。
2. **audit 不参与逐样本 routing。** audit camera 只在开封后用于科学判决，不决定某一行
   是否采用候选算子。所有 sample routes 都在 truth-field 和 audit evaluation 之前构造完成。
3. **决策对象做了提交指纹。** truth/audit evaluation 前的内存 routing SHA256 为
   `bbb6204e749575a2f458353992e31e9a4f75fa91b49e5cd156b19f358f22afea`；提交对象排除了
   `block_id`、`family` 和 `true_aperture_radius` 等报告字段。
4. **不直接拿真值或 family 标签选路。** direct truth fields、true radius 和 family labels
   均未用于 routing。`all_decisions_constructed_before_truth_and_audit_evaluation=true`。
5. **加入真删相机与双向 transfer。** 不只是看完整 block 的平均分，而是逐次删除 inner
   camera 后重新选参数，并要求 `outer_0→outer_1`、`outer_1→outer_0` 两个方向都过门。
6. **保留失败而不事后改锁。** post-open diagnosis 是只读诊断，状态为
   `V5C_POSTOPEN_NO_GO_DIAGNOSIS_NOT_A_NEW_LOCK`，不能拿诊断结果回头修改本轮协议再称为通过。

这些措施降低了明显的数据泄漏，但还不是密码学意义的封存：audit 在同一进程中于决策后打开，
没有独立审计进程。另有三项合成便利必须保留在边界里：metadata radius 是由真值派生的合成
proxy，noise whitening 使用 oracle nominal noise，support mask 使用与生成器匹配的 reaction
support。因此这仍是机制开发实验，不是部署级无泄漏证明。

## 3. 正式 NO-GO 数字

### 3.1 孔径与 `kappa` 选择失败在哪里

| 检查 | 正式结果 | 应怎样读 |
| --- | ---: | --- |
| 联合 `radius-kappa` CV 命中最近孔径 | `3/6 = 50%` blocks | 当前联合选择只恢复一半 |
| 固定 ridge 的最近孔径命中 | `5/6 = 83.33%` blocks | 固定复杂度后明显少错，但不是优越性证明 |
| operator-matrix oracle | `6/6` | 候选算子矩阵的孔径排序闭合 |
| clean truth-field oracle | `6/6` | 已知无噪声真场时排序闭合 |
| noisy truth-field oracle | `6/6` | 已知带噪真场时排序仍闭合 |
| 选到最大冻结 `kappa=0.001` | `6/6` blocks | 所谓最优点全在网格上边界 |
| best-CV profile 随 `kappa` 严格下降 | `6/6` blocks | 网格内没有观察到 interior optimum |
| 真删相机后的平均 radius 稳定率 | `0.375` | 删一台 inner camera 后经常换孔径 |
| 四次删除都稳定的 block | `0/6` | 没有 block 支持“共享孔径选择已锁稳” |
| 完整 block 描述门通过率 | `0/6 = 0%` | 每个 block 都有失败原因 |
| sample-excluded block gate 通过率 | `0/48 = 0%` | 每行删掉自己后，block gate 仍没有通过 |
| 最终 coverage | `0/48 = 0%` | 48 行全部回退，accepted count=`0` |

`kappa` 可以讲成“我允许重建场有多复杂”的刹车强度。当前 6 个 block 都选到测试范围内
最大的 `0.001`，而误差曲线到边界还一直下降，所以不能说我们找到了最佳 `kappa`；只能说
**当前评分持续偏爱更强的 ridge，`radius` 与 regularization complexity 被混在了一起。**

### 3.2 `0 coverage` 为什么不是成功

48 行中，只有 2 个 block、共 16 行真正改变了 metadata operator；其余 32 行明确 fallback。
对这 16 行做 post-open raw 诊断：

- field gain 平均 `+1.325%`，中位数 `+1.818%`，p10 `-7.183%`，范围
  `-11.116%` 到 `+14.832%`；
- `7/16 = 43.75%` 的 field accuracy 损失超过 `1%`；
- audit change 的定义是“正数更差”，其平均为 `-0.950%`、中位数 `-0.861%`，但
  `8/16 = 50%` 仍使 audit camera 变差；
- 两台 outer 中较差一台的 improvement，16 行平均为 `-9.348%`，中位数
  `-9.605%`，范围 `-48.330%` 到 `+14.403%`；
- 同时满足“最差 outer improvement 至少 `2%` 且 audit 不增加”的行数是 `0/16`。

严格门把这些候选全拒绝，说明**拒绝路径按规则工作**；但总是回退的方法没有展示任何实际
效用。正式报告里 mean selected audit change 是 `0.0%`，只是因为最终一行都没接受，不能翻译成
“风险为零”。accepted-only mean、p10、harm rate 和 audit increase rate 都没有可计算样本。

## 4. 三个容易误读的统计概念

### 4.1 48 行不是 48 个 IID 样本

这里有 48 个 nominal sample rows、6 个 `rig × radius` 配对 block、16 个 field-rig bundles，
但独立 rigs 只有 `2`。同一 rig 内的 radius blocks 会复用 fields 和 paired noise，各个 views
也是重复测量，不是新的独立实验单位。因此：

> **48 行不是 IID，不能用 `n=48` 的普通独立同分布假设去声称置信区间、显著性或风险上界。**

更诚实的最高层独立单位是 rig/session draw，而当前只有两套。

### 4.2 两台 outer camera 不是任意视角的风险保证

双向 transfer 的两个方向 coverage 都是 `0.0`、accepted count 都是 `0`。即使将来两方向都
通过，也只能说明这两台固定 outer cameras 之间的迁移表现；**两台 outer 既非 IID 抽样，也
不能给“任意新相机/任意新视角安全”提供校准风险界。**本轮 outer-audit Pearson `0.4984`
也只是描述性相关，不能当风险 bound。

### 4.3 “真删相机”与稳定率 0.375

真删相机不是从一张已算好的分数表里遮掉一列，而是每次真的把一台 inner camera 从选择流程
拿掉，再重新选择 `radius` 和 `kappa`。四次删除后，`kappa` 的平均稳定率是 `1.0`，因为全部
仍撞在 `0.001`；但 radius 的平均稳定率只有 `0.375`，且没有一个 block 四删全稳。讲人话：
**当前孔径答案很依赖“恰好放进了哪台相机”，还不够稳定到能锁成共享校准量。**

## 5. 本轮可以说什么，不能说什么

### 可以说

- 在当前合成 surrogate 上，有限孔径 bank 的 operator、clean-truth、noisy-truth oracle 均为
  `6/6`，所以失败已定位到当前联合 predictive-selection rule，而不是 bank semantics。
- 固定 ridge profiling 为 `5/6`，联合选择为 `3/6`，提示公平处理不同复杂度是下一步重点。
- 严格门拒绝了所有候选，coverage 为零；这证明当前门没有放过这些失败候选，不证明有用性。
- 当前结果足以判定 `NO_GO_JOINT_RADIUS_KAPPA_CROSSVIEW_SELECTION`。

### 不能说

- 不能说 v5c 已通过、已安全、零风险，或已经形成 confirmatory evidence。
- 不能把 48 行当作 48 个独立样本；也不能用 2 台 outer camera 声称 arbitrary-view safety。
- 不能称本轮为当前 family-OOD：`jet_shear`、`shock_cell` 参与了 inner selection。
- 不能说固定 ridge 已被证明优于联合 CV；`5/6` 只是 opened development 上的诊断线索。
- 不能说有限孔径 BOST 不可行，也不能把失败外推到 real BOS、OERF、非线性 ray tracing、CFD
  或任意真实采集 session。
- 不能说已经优于或否定 DeepONet、FNO、FFNO、NeRIF/TDBOST；本轮没有做这些公平比较。
- 不能根据 raw 平均 field gain 为正就称校准成功；16 个 changed rows 中仍有 `7/16` 明显伤害。

## 6. v5d 的五个算法备选

以下五项先在**已打开的 v5c development data**上公平比较，只用于方法开发；选定一项后再冻结，
不能把同一批 opened data 当成新确认集。

1. **GCV 或 UPRE + effective degrees of freedom：**先用有效自由度校准 `kappa`，再比较 radius，
   避免不同算子靠不同平滑强度取巧。
2. **Morozov discrepancy principle：**用外部标定的噪声水平选 `kappa`，让 residual 与预期噪声
   能量相符；不能继续使用部署时拿不到的 oracle noise 当真实可用输入。
3. **Equal-complexity / equal-effective-DOF radius score：**只在相同模型复杂度下比较不同 radius，
   把“孔径更对”与“正则更强”拆开。
4. **Genuinely nested inner-camera CV：**每个 outer inner-fold 内部先调 `kappa`，再在未参与调参
   的 inner camera 上给 radius 打分，杜绝同一 fold 同时调参和评分。
5. **Camera-consensus radius score + 预声明不稳定回退：**按相机分别形成 radius 证据，只有预先
   写死的共识和删相机稳定条件满足时才行动，否则回退。

下一次真正 first-open 应使用新的 `helical_plume`、`stratified_ignition` families、新 seed，
以及新的 rig/session draw。先在 opened v5c 上选规则，再冻结代码、阈值、tie-break 和失败条件。

## 7. 我接下来要补的数学与可执行练习

### 数学最小补课

1. **SVD 与 Tikhonov ridge。**能从 `A=UΣV^T` 推出 ridge filter factor
   `σ_i/(σ_i^2+λ)`，并解释为什么增大 ridge 会同时改变预测误差和可辨识的有效复杂度。
2. **Hat matrix 与有效自由度。**对线性 ridge 写出
   `H_λ=A(A^T A+λI)^{-1}A^T` 和 `df(λ)=tr(H_λ)`。
3. **GCV/UPRE。**理解 GCV 如何用 `n-df` 修正训练 residual；理解 UPRE 为什么需要可信的噪声
   方差，以及它和纯 residual 最小化的区别。
4. **Morozov 原理。**理解白化后 residual norm 应与噪声自由度同量级，而不是越小越好。
5. **嵌套交叉验证与层级抽样。**分清调参 fold、评分 fold、outer/audit；分清 row、view、block、
   rig/session 哪个层级可以近似看作独立抽样单位。

### 可执行练习

1. 在一个 `2×2` 对角病态矩阵上手算并画出 6 个冻结 `kappa` 的 filter factors、residual、解范数
   和 `df(kappa)`；必须能解释“residual 下降”为什么不等于“参数更接近真值”。
2. 对 v5c 六个 block 逐个重算 GCV、UPRE 和 Morozov 选择，输出统一 CSV：每行至少包含
   `block_id/radius/kappa/residual/df/score/selected`，并写单元测试检查 tie-break 固定。
3. 做 equal-DOF 配对图：每个 radius 找到最接近的公共 `df` 后再比较分数；若无共同复杂度区间，
   必须明确报“不可比较”，不能外推。
4. 手写真正 nested inner-camera CV 的索引测试：任何评分相机都不得出现在该 fold 的 `kappa`
   调参集合中；测试同时禁止 truth radius、family label 和 audit index 进入 routing features。
5. 做 rig-level bootstrap 的玩具练习，并亲自展示只有 2 rigs 时区间为何极不可靠；结论应是
   “需要更多独立 rig/session draws”，而不是输出一个看似精确的百分比。
6. 给 v5d 写预注册表：冻结算法、网格、噪声来源、共识门、fallback、双向 transfer、主指标、
   失败条件和代码哈希，再运行新 family/new seed/new rig first-open。

## 8. 要问何远哲的问题

1. 在真实 TDBOST/BOST 采集中，哪些变化可以合理视为独立的 rig/session draw？同一机架换角度、
   重复曝光、重新标定、重新搭台，分别算什么层级？
2. 是否能拿到独立标定的相机噪声、背景帧和读出噪声，用于 Morozov/UPRE，而不是 oracle nominal
   whitening？真实噪声是否明显异方差或相关？
3. 有效孔径在一个 session 内是否应物理共享？跨相机、跨视角、跨曝光时是否可能漂移？
4. 对真实实验，最重要的失败代价是三维场 L2、held-out reprojection、audit-camera 变差，还是
   某个物理量/积分量偏差？安全门应优先保护哪一个？
5. v5d 更适合先做传统正则选择的机制闭合，还是已有校准数据足以支持直接进入 NeRIF/TDBOST
   联合训练？如果进入网络，能否提供严格独立的 session 作为最终锁定集？
6. `helical_plume`、`stratified_ignition` 作为下一轮新合成 family，是否覆盖了课题组认为最危险的
   空间频率和形态变化？还缺哪类真实流场结构？

## 10. 一句话收尾

v5c 没有证明“跨视角联合选择成功”，但把失败定位得更清楚了：**前向算子排序是闭合的，当前
`radius-kappa` 评分却偏向最大 ridge，孔径对删相机不稳定，严格门只能做到全部回退。**v5d 的
任务不是扩大网络，而是先在等复杂度、可信噪声或真正嵌套的条件下，把“选正则”和“选孔径”
拆开，再用新的 family、seed 和独立 rig/session 做一次不能回头修改的 first-open。
