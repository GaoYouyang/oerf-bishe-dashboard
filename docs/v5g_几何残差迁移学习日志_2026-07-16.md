# V5G 几何残差迁移学习日志

日期：2026-07-16
实验：`v5g_geometry_residual_transfer_postopen`

## 1. 这次做了什么

这次实验研究一个很具体的问题：已知同一个物理系统在 source view（源视角）看到的观测，能不能利用相机几何的变化，预测 target view（目标视角）在几何改变后会出现怎样的残差变化？这里的“残差”就是观测减去当前模型预测之后剩下的部分。

实验没有重新使用 target observation，也没有使用 truth field。它先在 source views 上拟合一个残差场，再把这个残差场通过算子传播到 target views，比较几何改变前后的预测误差下降。实现入口是 `demo_t16_operator/view_transfer_geometry.py` 中的 `residual_field_transfer`。

本轮一共检查 24 个 changed samples，来自 3 个 changed-radius blocks 和 2 个独立 rigs。这里的样本不是独立同分布（`sample_rows_are_iid=false`），所以 24 个点不能被当成 24 次完全独立的实验。

## 2. 为什么 target operator 可以用，但 target observation 不能用

这是一条审计防火墙：

- **target operator 可以用**：operator 是由目标相机位置、几何参数和物理模型决定的“如何把场映射成观测”的规则。实验要测试的正是几何变化是否能解释残差迁移，因此必须知道 target operator。它类似于知道目标相机的镜头、位置和成像公式。
- **target observation 不能用**：target observation 是目标视角真正测到的数据。如果在预测 target 前看了它，就等于提前看答案的一部分，随后得到的“预测”不能证明迁移方法有用。代码因此只用 `values[:, source, :]` 形成 `source_residual`，target view 只在最后评估。

换句话说，本实验允许知道“目标视角怎样成像”，但不允许在本次预测计算中知道“目标视角实际看到了什么”。代码先冻结并哈希预测，再拼接 target 结果。不过，这组 audit 早已在上一轮打开，而且本轮 predictor family 是在打开后设计的，所以它只能叫 **post-open 机制诊断**，不能叫新的盲验证。

还要再澄清两个边界。第一，这不是“纯几何预测”：加权规则先读取 outer/source observation 上的实际误差变化，残差场方法也用 source observation 拟合残差，只是都不读取 target observation。第二，本轮 whitening 的 `noise_std` 由合成 clean inner-view 信号 RMS 推导，是 oracle σ；真实实验必须换成 flow-off/reference 重复帧估计，当前结果不能直接叫可部署 source-only 方法。

## 3. projection、Gram 和 leverage 是什么

这些量都是在问：两个视角虽然坐标不同，但它们对未知场的约束方向是否相似？

### 3.1 Projection：看约束方向的重合

把一个视角的观测矩阵记为 (A)。它的 row space（行空间）是所有观测对参数场施加约束的方向。`row_space_basis` 用 SVD 找到这个空间的一组正交基 (U)，相当于把重复或很弱的方向清理掉。

两个视角的 projection similarity 近似写成：

\[
s_{proj}=\frac{\|U_1^T U_2\|_F^2}{\sqrt{r_1r_2}},
\]

其中 (r_1,r_2) 是两个行空间的维数。值越大，表示两个视角“能看到的场方向”越重合；值小则表示一个视角提供的信息不能直接替代另一个视角。

### 3.2 Gram：看信息量和方向的整体相似性

观测矩阵的 Gram 矩阵是

\[
G=A^T A.
\]

它可以理解为一个简化的 Fisher information matrix：参数场中哪些方向被观测得强，哪些方向容易混淆，都被汇总在 (G) 里。代码用两个 Gram 矩阵的 Frobenius 内积计算 cosine：

\[
s_{gram}=\frac{\langle A_1^TA_1,A_2^TA_2\rangle_F}
{\|A_1^TA_1\|_F\|A_2^TA_2\|_F}.
\]

它不是比较两个观测值是否相等，而是比较两台相机对参数空间的“测量重点”是否相似。

### 3.3 Leverage：目标观测是否落在源拟合能支撑的范围内

用 source 的内层矩阵 (A_{inner}) 做带 ridge 的 Hessian：

\[
H=A_{inner}^TA_{inner}+\lambda I.
\]

对 target 矩阵 (A_{target})，每个目标测量的 leverage 近似是：

\[
h_i=a_iH^{-1}a_i^T.
\]

它回答的是：在 source 已经提供的约束下，第 (i) 个 target 测量有多“新”、多有预测影响力。总 leverage 是所有 (h_i) 的和，mean 是平均影响力，maximum 是最容易成为高影响点的测量。ridge 的 (lambda) 防止 source 对某些方向完全没信息时矩阵不可逆。

## 4. 残差场迁移的直觉和公式

直觉上，可以把 baseline field 看成当前模型对物理场的估计。source view 仍然有观测误差，于是：

\[
r_s=y_s-A_s f_{base}.
\]

这里 (y_s) 是 source observation，(A_s) 是 source operator，(f_{base}) 是 baseline field，(r_s) 是 source 残差。实验用 source 残差做一个带支持约束的 ridge 拟合，得到一个残差场 \(\hat f_r\)：

\[
\hat f_r=\arg\min_f
\left\|W_s(A_s f-r_s)\right\|_2^2+\lambda\|f\|_2^2.
\]

其中 (W_s) 是由噪声标准差产生的 whitening 权重，支持约束表示只允许拟合指定的场自由度。这个步骤不是凭空创造 target 数据，而是把 source 中尚未解释的结构压缩成一个场。

接着，在目标视角上先得到 baseline 的迁移预测：

\[
\hat y_{t,base}=A_{t,base}\hat f_r.
\]

然后用 candidate 与 baseline 的物理算子和场的变化修正它：

\[
\hat y_{t,candidate}
=A_{t,base}\hat f_r-
\left(A_{t,candidate}f_{candidate}-A_{t,base}f_{base}\right).
\]

最后把目标视角的 baseline RMS 与 candidate RMS 做比较：

\[
gain=100\times\frac{RMS_{base}-RMS_{candidate}}{RMS_{base}}.
\]

所以算法的核心假设是：source 中的残差有一部分是“可迁移的场结构”，而不是只属于某个相机、某次噪声或某个 rig 的偶然误差。它是一个物理启发的残差校正器，不是直接学习 target observation 的黑箱预测器。

## 5. 关键结果

本轮重点检查的候选 predictor 是 `source_residual_field_transfer_gain`；它没有被保留为可部署或可冻结方法。关键统计如下：

| 统计层级 | 结果 | 直观含义 |
|---|---:|---|
| sample sign agreement | **0.708**（17/24） | 24 个 changed samples 中，预测增益正负方向有 17 个和实际一致 |
| sample Pearson (r) | **0.135** | 点对点线性相关很弱，不能说预测了准确的增益大小 |
| block sign agreement | **1/3** | 3 个 changed-radius blocks 只有 1 个方向判断正确 |
| rig sign agreement | **0/2** | 两个独立 rig 都没有同时得到正确的平均增益方向 |
| sample MAE | 5.664% | 点级平均绝对误差仍然不小 |
| block MAE | 1.807% | block 平均后数值误差变小，但方向仍只对 1/3 |

两个 rig 的细节也提醒我们不能只看 sample 层面：`development_rig_a` 的预测平均增益为 1.231%，实际为 -0.781%；`development_rig_b` 的预测为 -0.168%，实际为 1.195%。因此 rig 层面是 0/2。

## 6. 为什么后两者否掉“方法冻结”

sample sign 0.708 很容易让人产生“方法已经成功”的印象，但它只说明在当前 24 个样本里，局部正负方向有一定命中。因为样本存在 block 和 rig 的聚集，局部命中可能来自某个视角、某个半径区间或某个实验条件，不能自动外推到新的 rig。

block sign 只有 1/3，说明一旦把相邻样本按 changed-radius block 汇总，方向稳定性明显不足。rig sign 为 0/2 更关键：两个独立 rig 都没有保持正确的总体方向。若一个迁移规则连独立装置层面的方向都不稳定，就还不能冻结为固定方法、固定阈值或毕业设计中的确定性结论。

报告因此给出 `NO_METHOD_FREEZE`：保留 residual-field transfer 作为 candidate architecture，但不授权 predictor ranking，也不授权 threshold sweep。也就是说，本次结果支持继续做受控验证，不支持宣布“这个方法优于其他方法”或“已经找到最优方法”。

## 7. 与 geometry-conditioned residual neural operator 的联系

当前代码可以看成一个小矩阵、参数较少、可解释的基线：它显式计算 operator 的几何变化，显式拟合 source residual field，再把修正传播到 target。未来的 geometry-conditioned residual neural operator 可以把这条思路推广为：

\[
\hat r_t=\mathcal{N}_\theta(r_s, A_s, A_t, g_s,g_t),
\qquad
\hat y_t= A_t f_{base}+\hat r_t,
\]

其中 (g_s,g_t) 是相机几何或其他几何条件，\(\mathcal{N}_\theta\) 学习“什么样的 source 残差应该如何随几何变化迁移”。物理 decoder 可以继续强制输出通过 (A_t) 或物理方程解释的结构，避免网络只记住 rig ID。

但神经算子不能绕过本次实验暴露的问题：它仍然需要新的 rig/session 做真正的独立验证，仍然不能在预测时偷看 target observation，也不能因为 sample sign 较高就声称 superiority。合理的下一步是冻结 source-residual encoder 和 physics decoder，在新 rig/session 上预先哈希全部预测，再进行 block 与 rig 层面的方向检验；若独立 target-view 的 sign consistency 仍然缺失，就应停止 RigCal 这条分支。

## 8. 学习练习

1. 用一个二维矩阵手算两个 row space 的正交基，并解释 projection similarity 为什么不等于观测值相关系数。
2. 对一个 (2\times2) 的观测矩阵计算 Gram 矩阵，比较改变一个观测方向和改变观测强度分别会怎样影响 Gram cosine。
3. 给定 (A_{inner})、(A_{target}) 和一个小的 ridge \(\lambda\)，计算一个 target measurement 的 leverage，并解释为什么 ridge 会降低数值不稳定。
4. 用一组 source residual 解带 ridge 的最小二乘问题，分别观察 whitening 和 support mask 对残差场 \(\hat f_r\) 的影响。
5. 设计一个有 2 个 rig、3 个 block 的小型交叉验证表，分别计算 sample sign、block sign 和 rig sign，并说明为什么不能只报告 sample sign。

## 10. 结论边界

这次实验的结论是：**残差场迁移值得作为候选架构继续验证，但证据不足以冻结方法，更不足以宣称 superiority。** sample 层面的 0.708 是一个值得学习的信号；低相关 (r=0.135)、block sign 1/3 和 rig sign 0/2 则说明它还没有表现出跨条件稳定性。
