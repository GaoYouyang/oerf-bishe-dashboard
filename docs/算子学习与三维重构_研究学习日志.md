# 算子学习与三维重构：阶段研究学习日志

> 更新时间：2026-07-15
>
> 这是一份阶段记录，不是成果宣传页。数字只对应当前仓库里的小规模、合成数据实验；“通过”或“失败”都按报告中的门槛原样记录。

## 先说现在做到哪一步

我们正在研究一个很具体的问题：从少量、带噪声的二维投影，能不能把三维场的大致形状重建出来。当前这三组实验还不是完整的真实 BOST/OERF 实验，而是一个受控的“试验台”：8×8 的小网格、5 层深度，训练场主要是 `gaussian` 和 `flame`，独立测试则加入了 `expanding_kernel`、`jet_shear`、`shock_cell`、`interacting_fronts` 等不同形状，并使用独立的锥形/弯曲射线生成器。

报告给出的边界很重要：目前只覆盖“规定的线性锥形/弯曲射线和反应场族”；没有 nonlinear refraction、CFD truth、OpenBOST、OERF、FNO、FFNO、DeepONet 或 Learned Primal-Dual 的比较。因此，这一阶段是在检查数值修正和实验规矩是否站得住，不是在宣布一个三维重构新算法。

## 今天实际尝试了什么

### 1. 第一版：`base_correction_independent_gate`

模型可以理解成：先得到一个基础重建，再让一个小网络分 4 个阶段提出“修正量”；修正量被限制在一定范围内，并配了一个 certificate（证书）检查，防止网络随意把结果推得过远。它不是凭空生成三维结构，而是在数值重建旁边做受限纠偏。

训练使用 3 个种子（91、92、93），100 个 epoch，参数量 4,411。结果报告的最终状态是 `INDEPENDENT_LINEAR_GATE_FAILED_OR_INCOMPLETE`。

- 在 validation、相对 `fixed_pg`：平均 relative L2 gain 只有 **0.069%**，最差 10% 样本的 gain 为 **0.005%**，超过 1% 的退化比例为 **0%**。
- 但报告把 `projected_bb` 选作确定性数值基线后，candidate 相对它反而是 **-25.414%**，最差 10% 为 **-37.820%**，退化比例 **100%**。
- 在最终 `independent_lock`、相对 `projected_bb`：平均 gain **-17.903%**，最差 10% **-31.424%**，退化比例 **100%**。
- certificate violation rate 是 **0%**，但 acceptance rate 也是 **0%**。这说明“没有越过证书边界”不等于“修正确实被接受”或“重建变好了”。

通俗地说：第一版的网络没有成功提供有用的纠偏；它至少暴露了一个问题，即如果只和较弱的 fixed PG 比，几乎不变的结果也可能看起来像一点点改善，而换成更强的数值基线后就站不住了。

### 2. 第二版：`pbb_base_correction_independent_gate`

这里的 `PBB` 是 projected Barzilai-Borwein。可以把它想成一种根据最近几步变化，自动估计“这一步应该迈多大”的投影梯度方法；“projected”表示每一步仍要投回允许的物理/数值范围内。它是数值优化基线或候选更新器，不是神经网络本身。

这版把模型宽度从 10 调到 12，参数量为 **5,989**，训练 120 epoch，种子为 101、102、103。结果比第一版健康很多，但最终门槛仍失败：报告仍标为 `INDEPENDENT_LINEAR_GATE_FAILED_OR_INCOMPLETE`。

- validation 相对 `fixed_pg`：平均 gain **22.808%**，最差 10% gain **12.548%**，超过 1% 退化 **0%**。
- validation 相对 `projected_bb`：平均 gain **2.368%**，最差 10% gain **1.267%**，退化 **0%**。
- independent_select 相对 `fixed_pg`：平均 gain **12.954%**，最差 10% gain **6.666%**，退化 **0%**。
- 但最终 independent_lock 相对 `projected_bb`：平均 gain **-0.102%**，最差 10% gain **-2.819%**，超过 1% 退化 **19.444%**。
- 三个 split 的 acceptance rate 和 descent gate rate 都是 **1.0**，certificate violation rate 是 **0%**。

所以，“验证集和选择集不错”是真实的；“独立锁定集也优于强基线”则不成立。这个失败很有价值，因为它把问题从“代码是否能跑”推进到了“选择阶段的改善能不能跨到真正冻结的数据”。

### 3. 第三版：`pbb_saturation_guard_independent_gate`

这是在 PBB 基础上加 saturation guard。通俗说，它监测每次纠偏相对于基础结果是不是过大；如果修正开始饱和或超过允许比例，就降低或拒绝这一步。它解决的是“更新太猛”的风险，不保证模型自动学到正确的三维物理。

参数量仍为 **5,989**，训练 120 epoch，种子为 111、112、113；最终独立锁定数据的噪声范围进一步到 **0.06、0.11、0.16**。

- validation 相对 `projected_bb`：平均 gain **2.084%**，最差 10% gain **0.175%**，退化 **0%**；平均 acceptance rate 约 **88.3%**。
- independent_select 相对 `projected_bb`：平均 gain **0.525%**，最差 10% **-0.616%**，超过 1% 退化 **7.143%**。
- independent_lock 相对 `projected_bb`：平均 gain **0.281%**，最差 10% **-0.831%**，超过 1% 退化 **7.5%**。
- certificate violation rate 仍为 **0%**，descent gate rate 为 **100%**；但最终 gate 仍没有通过，因为要求平均 gain 至少 2%、p10 gain 不得为负、退化率不超过 5%。
- history 还显示最后一个种子在训练后段 acceptance 从 **0.80、0.60 降到 0.30**，说明“继续迭代”并不自动等于更好。

它比第二版的最终锁定结果更接近稳定：没有大幅崩坏，但平均优势只有 **0.281%**，仍不足以称为可靠改善。guard 的价值首先是控制风险，而不是制造显著增益。

## 名字翻译成“人话”

| 名字 | 它大致在做什么 | 不应误解成什么 |
| --- | --- | --- |
| `base_correction` | 在基础重建上加受限的分阶段纠偏 | 不是已经学会了真实三维物理 |
| `fixed_pg` | 步长固定的投影梯度更新 | 不是最强基线，只是一个稳定参照 |
| `projected_bb` / PBB | 根据历史变化自动调步长，并把结果投回允许范围 | 不是神经网络创新，也不是天然抗噪 |
| `FISTA` | 带“动量”的梯度更新，试图更快下降 | 更快下降不代表独立测试更准 |
| `saturation_guard` | 修正过大时踩刹车 | 踩刹车不等于方向正确 |
| certificate | 检查修正是否越过预设安全边界 | 证书通过不等于物理真实性得到证明 |
| `validation` | 用来比较候选方案的数据 | 不能当最终泛化证据 |
| `independent_select` | 与训练分开的数据，用来选方案 | 仍参与了方案选择，不能冒充最终测试 |
| `independent_lock` | 方案冻结后才看的数据 | 当前也只是合成独立生成器，不是真实实验 |

## 为什么失败仍然有价值

1. **把“看起来有效”和“经得住独立数据”分开了。** 第二、三版在 validation 有约 2% 以上的 PBB 对比 gain，但 lock 集没有复制出来，说明早期数字不能直接升级成论文结论。
2. **找到了强基线的影响。** 第一版相对 fixed PG 只有 0.069% 的微小改善，但相对 projected BB 大幅变差。以后若不锁定数值基线，模型优势会被比较对象掩盖。
3. **检查了安全性和有效性是两件事。** 三组 certificate violation 都是 0%，但第一版 acceptance 是 0%，且结果很差。安全边界只是底线，不是效果证明。
4. **保留了可重复的失败证据。** 三组报告都有配置快照、历史、样本指标和 checksum；这让后续可以定位是数据生成、停止规则、容量、噪声，还是模型结构的问题，而不是凭印象改网络。

## 当前不能声称什么

- 不能声称已经完成了三维重构算法，更不能声称在真实 BOST/OERF 数据上有效。
- 不能声称 CG-PDNO、PBB 或 saturation guard 是创新算法；当前只是受控合成试验台上的候选机制和数值基线。
- 不能把 validation 的 **22.808%** 或 **21.091%** gain 写成最终泛化提升；最终 lock 集相对强 PBB 只有 **-0.102%** 和 **0.281%**。
- 不能声称 certificate 通过就满足物理正确性；当前只证明没有违反这份人工设定的数值证书。
- 不能把当前独立生成器等同于 CFD、真实火焰、真实相机噪声或真实折射过程。
- 不能把 L2、梯度 L2、front F1 的一次小规模结果外推到所有视角、噪声、分辨率和三维结构。

## 2026-07-15 第四轮：`pbb_ensemble_selective_gate`

这一轮尝试的是“多个成员先各自提出修正，只有看起来可靠时才采用”的选择式集成。结果目录是 `demo_t16_operator/results/pbb_ensemble_selective_gate/`。报告状态仍是 `INDEPENDENT_LINEAR_SELECTIVE_ENSEMBLE_GATE_FAILED_OR_INCOMPLETE`。

### 五头到底是什么意思

这里的“五头”不是五次完整的三维物理重建。5 个成员各有自己的纠偏参数，训练种子是 121、122、123、124、125；每个成员约 5,989 个参数，总参数量约 29,945。它们先共享同一条 PBB 物理轨迹：4 次 forward 和 4 次 adjoint 只算一遍，然后让 5 个纠偏头在这条共同的轨迹上给出不同修正，再用成员之间的分歧估计“不确定性”。报告明确记账为：`candidate_shared_forward=4`、`candidate_shared_adjoint=4`、`correction_head_passes=5`，并且 `physical_trajectory_repeated_per_head=false`。

讲人话：先把“观测经过物理算子会怎样”这件昂贵的事情算一次，再让五个小专家讨论怎么改；不是把同一个物理模拟无理由地跑五遍。因此它在计算预算上是一个有意义的控制设计，但“五个头”本身不等于五倍证据，也不等于五倍泛化能力。

### 精确谱范数改了什么

之前的 certificate 使用 power iteration（幂迭代）估计 Lipschitz/谱范数，再乘一个安全因子。可以把它理解成：反复试探矩阵最容易放大的方向，经过有限次试探后得到一个经验上界。它有用，但仍然是“迭代得到的估计”，不能把它说成严格精确的矩阵范数。

这一轮改成了 `exact_small_matrix`：对于当前很小的矩阵，直接计算其精确谱范数，安全因子也从旧实验的较宽松经验设置改为 `1.001`。因此这轮的 certificate 检查在数学口径上更干净，报告中的 `certificate_violation_rate` 是 **0%**。但这只说明修正没有违反当前的小矩阵安全边界，不说明修正一定比 fallback 好，也不说明它符合真实三维物理。

### 选择阈值为什么没有可行值

阈值表示“成员之间意见要一致到什么程度，才允许采用集成修正”。报告扫描了 **41** 个候选阈值；选择门槛要求至少覆盖 **20%** 的样本、最差 10% 的 gain 不低于 **0%**、超过 1% 的退化率不高于 **5%**。

但没有任何阈值同时满足这三个条件。结果文件把阈值设为 **-1.0**，选择理由是 `no_feasible_select_threshold_abstain_all`，也就是全部 abstain（不采用集成修正）。所以不是“模型判断所有样本都不确定”这个物理结论，而是“在当前选择集上，找不到符合预设门槛的可靠覆盖区域”。

选择集的真实数字是：candidate 与 PBB baseline 的 relative L2 都是 **0.4461926**，平均 gain **0%**，p10 gain **0%**，退化率 **0%**，coverage **0%**；gradient relative L2 都是 **0.5656898**，front F1 都是 **0.7647003**。这些相同数字来自严格回退，不是集成模型把误差神奇地修到了一样好。

锁集使用了全新的 family-held-out 场：训练/选择侧主要是 `gaussian`、`flame` 以及 `expanding_kernel`、`jet_shear`、`interacting_fronts`、`shock_cell`，最终锁集则换成 **`helical_plume` 和 `stratified_ignition`**，并使用更强的三维弯曲/锥形射线和更高噪声范围 **0.07、0.12、0.18**。这是更严格的新锁集，不是把选择集换个随机种子再跑一次。

锁集的真实数字仍然是严格回退 PBB：candidate 与 baseline relative L2 都为 **0.5018178**，平均 gain **0%**，p10 gain **0%**，退化率 **0%**，coverage **0%**；gradient relative L2 都为 **0.6504059**，front F1 都为 **0.6310366**。因此本轮没有得到“集成修正有效”的证据，也没有得到“集成修正有害”的证据；得到的是一个明确的拒绝策略：当前证据不足时，系统不采用修正。

### 红队发现：PBB gate 的保证范围比想象中窄

这轮还澄清了一个容易写错的逻辑：PBB gate 只保证相对 **shared base** 的某个下降条件，也就是检查更新后是否比共享的基础轨迹更好一些。它**不保证**结果一定优于最终 fallback（这里是确定性 `projected_bb`）。

这两个比较对象不是一回事：

- `shared base`：五个头共同接收到的基础状态/方向，gate 可用它检查“这次修正有没有把共享状态往下降方向推”。
- `fallback PBB`：当阈值不可信或 gate 不通过时，最终采用的保底数值解。

如果 gate 只对 shared base 负责，却把结果拿去和更强的 fallback 比，就可能出现“gate 通过了，但最终并没有优于 fallback”的逻辑漏洞。本轮通过全 abstain 避免了这个漏洞，但这不等于模型通过了性能门槛；它只是诚实地退回 PBB。

### 这一轮失败为什么仍然有价值

1. **选择式集成没有被包装成成功。** 覆盖率是 **0%**，gain 是 **0%**，这正是报告应呈现的结论，而不是把回退结果写成“稳定提升”。
2. **family-held-out 锁集更接近真正的外推检查。** `helical_plume` 和 `stratified_ignition` 没有出现在选择集里，至少避免了对已见 family 的过度乐观解释。
3. **计算预算和物理轨迹被记清楚了。** 五个头共享 4 次 forward/4 次 adjoint，后续比较时不会误把五个纠偏头算成五套完整物理实验。
4. **红队修正了保证范围。** “相对 shared base 下降”不能偷换成“优于 fallback PBB”；这个区分会直接影响论文里的 gate 定义和结论措辞。
5. **精确小矩阵谱范数减少了证书口径的不确定性。** 但它没有制造效果提升，反而让我们更清楚地看到：安全性证明和泛化效果仍是两条独立证据链。

### 第四轮之后不能声称什么

- 不能说五头集成提升了精度；本轮选择集和锁集相对 PBB 的 gain 都是 **0%**。
- 不能说覆盖率 0% 证明所有新样本都不适合学习；它只证明当前选择阈值规则找不到可行覆盖。
- 不能说 exact small-matrix certificate 证明了物理正确性；它只是比旧的 power-iteration 经验上界更精确地检查了当前小矩阵约束。
- 不能说 PBB gate 保证优于 fallback；当前它只检查相对 shared base 的下降条件。
- 不能说 family-held-out 锁集已经代表真实实验；它仍是规定的线性弱偏折弯曲/锥形生成器，不是 CFD、OpenBOST/OERF 或真实传感器。

### 第四轮后的下一步

## 2026-07-16 第五轮：第一次真正把 forward-model mismatch 放进门槛

前四轮都默认“生成数据的算子就是重建使用的算子”。这适合检查优化器和网络修正，却
绕开了真实 BOST 很麻烦的一件事：相机并不是理想针孔，有限孔径、景深、光路弯曲和标定
误差会让真实成像算子 `A_truth` 与电脑里的 `A_reconstruction` 不一样。

第五轮 `v5a_blind_aperture_calibration` 专门检查这个问题。程序把每个 detector pixel
替换成一组确定性的圆盘子光线；生成观测时用更密的子光线和路径采样，重建时只给 5 个
较粗的候选孔径算子。它仍然只是线性弱偏折小模型，不是完整 cone-ray 或真实光学。

### 冻结前抓到的一个真实泄漏

第一次审计时发现：虽然 audit camera 没进入 view mask，但噪声尺度的计算曾把 audit
camera 的 clean signal 也算进去。这样重建端会通过 `noise_std` 间接“闻到”一点 audit
camera 信息。这个通道很窄，也不能假装不存在。

我把噪声尺度改成只用真正参与重建的相机，并增加了一个测试：把 audit camera 的信号
放大一百万倍，噪声尺度仍必须完全不变。旧开发结果因此作废并重跑。

### 为什么要先把失败标准发到 GitHub

修复后先建立预注册提交 `90ff961`，再打开锁定集。预注册写死了：

- 最强等预算基线是 60 步针孔 FISTA；
- 候选的 5 个算子 probe 共花 30 次 forward/adjoint，最终重建再花 30 次；
- 开发集选中的方法是 `full_residual_hard`，阈值为 `0.0218204`；
- 锁集必须同时满足 mean ≥2%、p10 ≥0、`>1% harm ≤5%`、audit reprojection 不上升；
- 失败后不能改阈值再把同一个 lock 叫“首次测试”。

这个流程的意义是：结果出来以后，即使只差一点，也不能移动球门线。

### 首开数字

锁定集换成未见的 `helical_plume` 与 `stratified_ignition`，未见真实孔径
`0.06 / 0.10 / 0.14`，另一套相机角度和锥束参数，以及不同的 audit camera。

- 针孔失配惩罚 **3.144%**：说明这次 mismatch 不是零；
- 相对等预算针孔 FISTA，候选平均 gain **+1.753%**，低于 2% 门槛；
- p10 gain **-17.784%**；
- 超过 1% 的退化率 **25%**；
- audit-camera 重投影平均变化 **-0.439%**；
- 五项门槛只过两项，最终状态是 `FAILED_OR_INCOMPLETE`。

最容易误写的是：平均误差确实从 `0.45170` 降到 `0.43522`，但这不能掩盖有四分之一
样本明显变差。论文如果只报平均值，就会把一个危险的选择器写成改善。

红队还指出，原先 `25%` 用 36 个全部样本作分母，12 个回退样本以 0 gain 稀释了风险。
真正采用候选的 24 个样本里有 9 个明显变差，条件伤害率是 **37.5%**，accepted-only
p10 是 **-24.779%**。下一版必须同时报告总体风险和 `risk | accept`，不能只看前者。

### 为什么“自己的残差最小”不等于“孔径猜对了”

未知量不只有孔径，还有整个三维场。某个错误孔径算子可以通过重建出另一种三维结构，
把可见相机拟合得很好。因此，小 residual 可能来自两种解释：

1. 光学孔径确实更接近真实值；
2. 三维体场被扭曲后，替错误孔径吸收了误差。

锁后诊断显示第二种解释占了主导：`34/36` 个样本被选到孔径库两个端点；螺旋羽流
平均被估成 `0`，分层点火平均被估成 `0.156`。接受样本里 confidence 与真实 gain 的
秩相关只有 **-0.0026**，基本没有风险排序能力。

讲人话：算法以为自己在看镜头，实际上主要在认火焰/流场长什么样。

### 这一轮增加了什么真正的知识

1. **把 inverse crime 再往前推了一步。** 不再只换随机场，而是让真实与重建算子明确
   不同，并保存各自哈希。
2. **证明了 raw residual blind calibration 的具体失败模式。** 它会把 morphology 与
   aperture 混在一起，而不是简单“噪声大所以不准”。
3. **明确下一步应该先做 identifiability。** 在允许三维场一起改变后，如果孔径扰动
   完全能被体场变化吸收，就应该 abstain，不能硬估孔径。
4. **没有把有限孔径或 profile-Fisher 包装成原创。** cone-ray、blind calibration、
   profile likelihood、Fisher information、Schur complement 和 selective prediction 都
   有前人工作；可能的新意只能来自 BOST 专属的可观测性证据、尾部风险和真实迁移。

### 下一条 v5b：先问“孔径可不可辨识”

最值得验证的是 profile-Fisher / nuisance-orthogonal score。把体场 `x` 当 nuisance，
把孔径 `r` 的测量方向 `J_r=(∂A/∂r)x` 投影掉 `J_x=A(r)` 能解释的部分。剩下的能量
如果很小，就表示“改变孔径”和“改变三维场”在当前相机下分不开，应回退强基线。

它还只是研究假设。v5a lock 从现在起只能作为 v5b 的开发诊断；v5b 必须另开新 family、
新几何、新连续孔径和新锁集。完整公式、对照和师兄问题已写入
`v5a_blind_aperture_first_open_review.md`。
