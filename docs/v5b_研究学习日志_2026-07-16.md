## 1. 今天在问什么

有限孔径 BOST 的观测可以简写为

\[
y_s=A_{\mathrm{truth}}(r_*)x_s+\epsilon_s.
\]

`x_s` 是每个未知三维场，`r_*` 是同一相机机架或采集块共享的有效孔径。我们真正想问的是：

> **当多个形态不同的未知体场共享一个低维光学参数时，能否把“体场变了”与“孔径变了”分开；如果分不开，能否可靠地拒绝校准并回退到稳健基线？**

v5a 是让每个场单独用 residual 猜孔径；v5b 改为“多场共享孔径 + profile objective + Schur/profile 可辨识性 + outer camera 回退”。这是一个待证伪的方向，不是已经成立的新算法。

## 2. 从 v5a 到 v5b，真实进了哪一步

### v5a 告诉我们：单场最小残差会认错东西

v5a 首次明确分开 `A_truth` 和 `A_recon`，并在预注册后打开 36 个锁定样本。结果不是“差一点就成功”，而是尾部风险明显失败：

| v5a 预注册项 | 结果 | 判决 |
| --- | ---: | --- |
| 针孔失配 penalty | `3.144%` | 通过，说明失配不是零 |
| 平均 L2 增益 | `+1.753%` | 未达 `2%` |
| p10 增益 | `-17.784%` | 失败 |
| 退化超过 `1%` 的总体比例 | `9/36 = 25.0%` | 失败 |
| audit-camera 预注册平均口径 | `-0.439%` | 通过 |

五项只过两项，v5a 的总判决是 `FAILED_OR_INCOMPLETE`。锁后红队还发现，不能用大量“回退=零增益”的样本稀释候选方法的真实风险：24 个接受样本中有 9 个伤害超过 `1%`，所以

\[
P(\mathrm{harm}>1\%\mid\mathrm{accept})=9/24=37.5\%,
\]

接受样本的 p10 是 `-24.779%`。这比总体平均 `+1.753%` 更能说明问题。

锁后诊断也给出了“形态/算子混淆”的线索：

- `34/36` 个样本把孔径选到候选网格的两个端点；
- 24 个接受样本中，`23/24` 的孔径误差至少是 `0.06`；
- confidence 与真实 gain 的 Spearman 相关只有 `-0.0026`；
- 最差样本把真实孔径 `0.14` 判成 `0`，L2 gain 为 `-30.24%`。

这些数字支持“简单 residual margin 没有在排序安全性”，但锁后分析不能单独证明唯一因果机制。

### v5b 做到了：把新假设变成可运行的小模型检查

v5b 已有一次 development-only pilot，而不再只是口头方案。它包含：

- `4 families × 2 fields × 2 rigs × 3 true radii = 48` 行配对样本；
- 16 个 field-rig bundles，最高共享层只有 6 个 `rig × radius` blocks；
- 真实孔径为 `0.055/0.095/0.135`，候选库为 `0/0.04/0.08/0.12/0.16`；
- inner views 拟合场与孔径，outer views 只用于接受判决，audit view 只报告；
- 显式 ridge profile 求解、共享孔径、metadata prior、profile-Schur 量、oracle 和辨识性扫描都已经产生机器可读结果。

这是工程和实验设计上的进展：我们现在可以看见 profile curve、孔径误差、边界命中、oracle headroom 和拒绝覆盖率，而不是只看一个平均 L2。

## 3. 为什么先做 oracle，而不是先加网络

oracle 使用实际部署时拿不到的真值信息，它不是可部署基线，而是一把“实验设计尺子”。先做 oracle 有三个原因。

1. **检查孔径语义。** 已知真实体场时，哪个候选算子最能解释观测？与 truth operator 矩阵距离最近的又是哪个？如果这两个最基础检查都不指向真实孔径的最近候选点，先应查 renderer、离散化、尺度和候选库，不应训练更复杂的选择器。
2. **检查上限（headroom）。** 已知最近孔径甚至已知真实算子时，在同一重建器、正则和指标下是否真的比 pinhole 好？如果这个上限不存在，那么“估准孔径会改善 L2”这条因果链尚未建立。
3. **防止把其他偏差当成校准成功。** 错孔径可能偶然带来更强的平滑或更合适的数值尺度，从而让 L2 变好。这种“赢”说明重建偏差改变了，不说明孔径被辨认了。

所以正确顺序是：**oracle closure → 传统小模型机制 → 新 development → 预注册 lock → 大模型/神经算子。**

| 方法/诊断 | 平均孔径 MAE | 命中最近候选点 | 边界命中 |
| --- | ---: | ---: | ---: |
| per-scene profile | `0.095` | `0%` | `100%` |
| rig-shared profile | `0.095` | `0%` | `100%` |
| metadata-anchored RigCal profile | `0.095` | `0%` | `100%` |
| metadata nearest | `0.020` | `50%` | `16.7%` |
| oracle nearest bank | `0.015` | `100%` | `0%` |
| truth-field residual oracle | `0.020` | `50%` | `16.7%` |
| closest truth-operator matrix | `0.0233` | `16.7%` | `33.3%` |

6 个 block 中，per-scene、shared 和加 metadata prior 的 profile 都选了孔径 `0`。但这里的场
nuisance 实际使用 320 个未知体素拟合 160 条 inner measurements，不能把端点失败单独归因于

更早的 oracle sanity check 也没闭合：用真实体场算 residual，只有 `3/6` blocks 选中最近孔径；直接比较 truth/reconstruction operator 矩阵距离，只有 `1/6` 选中。这说明当前首先要查候选算子库与 truth renderer 的孔径对应关系，而不是调 accept threshold。

### 4.2 profile-Schur 没有给出可用的可辨识性信号

- 6 个 block 的 median profile retained fraction 只在 `0.000320` 到 `0.000708` 之间；
- 开发配置的接受门槛是 `0.005`，所有 block 都低于它；
- 所有 block 同时撞到孔径下边界，而且多数与 metadata 冲突；
- 因此 `block_identifiable = false` 在 `6/6` blocks 上成立。

锁后不存在，这里是 development 诊断。一个 210 行的事后扫描又检查了 inner view count `3/4/5/6` 和 ridge `0.01/1/100/10^4/10^6`：

- 所有扫描设置的 shared nearest-bank match 仍是 `0%`；
- 所有设置的 boundary-hit 仍是 `100%`；
- 强行把 ridge 增大时，median retained fraction 最高可到 `0.9924`，但孔径恢复一次也没改善。

这是一个很有价值的反例：**regularized profile 数值变大，不等于数据真的更能辨认孔径。**

### 4.3 `0/48` 接受不是“零风险成功”

由于所有 block 都被判为不可辨识，v5b 安全层对 48 行样本全部回退：

- coverage = `0/48 = 0%`；
- accepted count = `0`；
- overall gain、p10 和 harm 在“全部回退”口径下都是 `0`；
- accepted-only mean/p10/harm 全部不可计算，报告的正确解释是 `INCONCLUSIVE_FOR_ACCEPTED_CONDITIONAL_RISK`。

这说明“拒绝不可辨识更新”的代码路径工作了，但没有说明方法能在有用 coverage 下控制风险。

### 4.4 oracle 没有显示出 L2 headroom

| 重建算子 | 平均 relative L2 | 相对 pinhole 的 L2 gain |
| --- | ---: | ---: |
| pinhole | `0.676545` | 基准 |
| oracle nearest bank | `0.677367` | `-0.121%` |
| oracle true operator | `0.683027` | `-0.958%` |

这并不表示“真实有限孔径算子没用”。它表示：在**当前合成模型、固定 ridge=`1`、当前 support 和 relative L2 口径**下，“选对算子单独带来三维 L2 改善”还没有被证明。

同时，oracle true operator 的 outer RMS 由 pinhole 的 `9.433` 降到 `9.097`，audit RMS 由 `7.675` 降到 `7.314`，但场 L2 反而更差。这正好证明“观测拟合更好”与“未知三维场更准”是两件事。

### 4.5 修正版 M1：尺子闭合了，安全门仍失败

M1 不是在旧结果上换一句解释，而是同时改了两件会改变科学语义的东西：显式二值支撑域从
错误的 `320/320` 改为阈值冻结的 `96/320`；重建 renderer 只能选择先通过独立 Stage 0 的
`path=35, aperture=41` setting。Stage 0 共检查 72 个 rig-radius-renderer 组合，最终只有两个
setting 能在 native/shared-scale/gain-profiled 三种距离下同时 `6/6` 命中最近孔径。

M1 的校准证据阶梯如下：

| 检查 | 最近孔径命中 | 它在排除什么 |
| --- | ---: | --- |
| closest truth-operator matrix | `6/6` | 候选算子库的孔径排序错误 |
| clean truth-field residual oracle | `6/6` | 无噪声真场下的数据排序错误 |
| noisy truth-field residual oracle | `6/6` | 当前噪声单独打乱排序 |
| rig-shared SSE-only | `5/6` | 未知场 nuisance 后仍保留大部分孔径信号 |
| fixed-ridge regularized profile | `4/6` | 固定 penalty 会改变算子之间的排序 |

事后机制扫描还发现：使用 6 个 inner views、`lambda=1` 时可以 `6/6`，但 3--5 个 views 或
其他 ridge 并不稳定。这个结果只能生成下一轮假设，不能回头修改 M1 的开发配置并自称通过。
最合理的新问题不再是“孔径有没有信号”，而是：**怎样在不同算子、不同视角数之间公平选择
正则强度，并让选择标准不读取最终审计相机。**

M1 当前门接受 `11/48=22.92%`。已接受样本的内层 field L2 平均改善 `9.36%`，但独立 audit
view 中 `5/11=45.45%` 反而变差，最坏 audit RMS 增加 `32.11%`。总体平均下降不能掩盖这个
尾部：对于“系统只有在安全时才采用校准”的主张，45% 的 accepted audit increase 足以否掉
当前门。它不是零风险，也不是“再调一点阈值就能发表”。

## 5. 失败意味着什么，又不意味着什么

### 这次失败意味着

1. v5a 的单场 residual margin 不能可靠排序校准风险。
2. 仅把孔径改为多场共享，并不会自动创造可辨识信息；v5b 当前仍全部选中端点 `0`。
3. 当前 candidate bank/truth renderer 的孔径对应连 oracle 都未完全闭合，应先修复机制检查。
4. profile-Schur retained fraction 会被正则强度显著影响，不能在没有孔径恢复和风险排序证据时单独作为“可辨识”证明。
5. 现在最值得做的不是调阈值或上神经网络，而是先完成 oracle closure 和重建上限检查。

### 这次失败不意味着

1. 不意味着现实 BOST 中的有限孔径校准一定不可行；当前只是线性弱偏折、8×8×5 显式 ridge 小模型。
2. 不意味着 Schur complement、profile likelihood 或共享校准的数学错了；它们是标准工具，失败的是当前模型与数据没有提供足够、可用的区分方向。
3. 不意味着 coverage `0%` 就证明安全；没有接受样本，就没有 accepted conditional risk 证据。
4. 不意味着 raw gain 为正就证明校准成功。当前选到错误端点 `0` 的 raw candidate 相对 metadata fallback 平均 L2 gain 仍有 `+0.817%`，这是增益混淆的直接例子。
5. 不意味着已经否定或超越任何神经算子；本轮根本没有做这种公平对比。

## 6. 今天学到的数学

### 6.1 Profile objective：先让每个场尽力解释，再比孔径

对每个候选孔径 `r`，先解每个场的内层问题：

\[
\hat x_s(r)=\arg\min_x
\left\|W_s^{1/2}(y_s-A(r)x)\right\|_2^2+\lambda R(x).
\]

然后再比较共享孔径的外层目标：

\[
\Phi(r)=\sum_s
\left[
\left\|W_s^{1/2}(y_s-A(r)\hat x_s(r))\right\|_2^2
+\lambda R(\hat x_s(r))
\right]
+\gamma\|r-r_{meta}\|^2.
\]

讲人话：不能把未知场假装成已知；对每个孔径，都先让体场做到它能做的最好，再看哪个孔径还留下更小的目标值。

但 `argmin Phi(r)` 只说明“它是当前候选集、正则和近似模型里的最小值”，不说明它就是物理真孔径。旧 M0 的 6 条 profile curve 都单调选到 `0`；修正后的 M1 不再全撞端点，却仍从
SSE-only 的 `5/6` 降到 regularized profile 的 `4/6`，两者共同展示了这个边界。

### 6.2 Schur complement：把“改场就能解释”的方向扣掉

在当前解附近，小变化满足

\[
\delta y\approx J_x\delta x+J_r\delta r.
\]

`J_x` 表示改体场如何改观测，`J_r` 表示改孔径如何改观测。把 `x` 当成 nuisance 消去后，孔径的 regularized local profile curvature 可写成

\[
G_{r\mid x}=G_{rr}-G_{rx}G_{xx}^{-1}G_{xr}.
\]

如果 `J_r` 造成的观测变化几乎都能用某个 `J_x delta x` 模仿，扣完后剩下的信息就很小，孔径在局部上不可辨识。

今天还学到一个重要限制：加入 ridge 后，这个量更准确地叫 **regularized local profile curvature**，不应随便写成纯 data Fisher。强 ridge 会让体场更难去吸收孔径方向，从而人为抬高 retained fraction；它不一定是数据新提供了信息。

### 6.3 增益混淆：估参数、拟合观测、重建体场是三个问题

今天最重要的数学/实验认识是：

\[
\text{孔径误差小}
\not\Rightarrow
\text{held-out residual 小}
\not\Rightarrow
\text{三维场 L2 小},
\]

反向也都不成立。

- 错孔径 `0` 可以相对 metadata fallback 得到 `+0.817%` raw L2 gain；
- 真实算子可以改善 outer/audit residual，却在当前 ridge 下恶化体场 L2；
- 因此，一个方法的 L2 gain 可能来自更合适的隐式正则、算子尺度或 solver bias，而不是来自正确光学校准。

所以以后的结果表必须分栏报告：**aperture error、field error、inner/outer/audit residual、coverage/conditional harm**，不能用一个“平均 gain”代替全部因果链。

## 7. 下一步实验

### 实验 1：先闭合 oracle/operator bank（当前最高优先级）

1. 先在无噪声、固定真实体场上，对每个 rig 和每个 true radius 画出：
   `truth-field residual vs candidate radius` 与 `operator distance vs candidate radius`。
2. 分两步做：先让 truth/reconstruction 使用同精度积分检查代码语义，再恢复不同子光线/路径采样检查离散化失配。
3. 检查不同孔径下算子的 norm、信号尺度和单调性，但不能为了让结果好看而事后任意归一化。
4. 通过标准应先写死：真值场 residual 和 operator-distance 的最优候选应与预期的近邻半径一致，否则不进入 gate 调参。

本轮已经开始把它实现成独立的 `forward-model convergence gate`：同时比较 renderer 各自
归一化、共享物理尺度和剖面掉一个全局增益后的 operator distance。第一轮单机架粗扫显示，
9 条孔径子光线时 `0/3` 真孔径命中最近候选；提高到 29 条后部分积分组合可达到 `3/3`，
但 path/aperture 积分存在交互，不能只凭“采样更多”口头认定收敛。最终 72 行两机架实验找到
两个安全 setting：`(29,41)` 与 `(35,41)`；二者在三种距离口径下都为 `6/6`。M1 冻结使用
`(35,41)`。这只证明当前弱偏折离散 renderer 的数值排名，不证明真实 BOST 或绝对幅值可辨识。

### 实验 1B：把增益 gauge 明写出来

使用 `y=gA(a)x+ε` 检查三件事：

1. 固定物理单位和外部幅值锚时，`a` 是否可恢复；
2. 只允许以 1 为中心、由标定误差给出的窄增益 nuisance 时，`a` 的 profile 是否稳定；
3. 放开自由全局增益后，明确展示 `(g,x)→(g/c,cx)` 的尺度歧义，禁止把不可辨识的绝对场
   幅值包装成算法成果。

### 实验 2：单独检查“正确算子的重建上限”

1. 在 development 上对 pinhole、metadata nearest、oracle nearest 和 oracle true operator 做公平的 ridge/停止扫描；
2. 正则应按事先规则选择，不能让某个算子固定吃到更不合适的 `lambda=1`；
3. 同时报告 field L2、outer/audit RMS 和计算成本；
4. 只有 oracle 在多个 block/场上稳定显示出可用 headroom，才值得研究如何估计这个算子。如果它只改善观测一致性而不改善体场，就要重新定义主终点，不能偷换。

### 实验 3：在可控 toy 中建立可辨识性

1. 从“已知场”逐步放开到低维场、support-constrained 场、完全未知场；
2. 分别增加视角、独立 PSF/reference 元数据和场数，找出哪种新信息真的打破 `J_r` 与 `J_x` 的重合；
3. data-only curvature、metadata curvature 和 regularized total curvature 分开报告；
4. 要求可辨识分数与孔径 MAE、boundary hit 和 harm 具有稳定排序，不再只看分数绝对值。

### 实验 4：闭合后再比较单场与共享校准

沿用配对 `field × family × radius × rig` 设计，但把评价重点放在：

- shared 是否真的降低 aperture MAE 和 boundary-hit rate；
- 估计孔径是否不再随 family 变化；
- profile-Schur 是否能排序误差和伤害；
- outer camera 是否能在非零 coverage 下降低 accepted conditional harm。

只有开发集上 Gate A 出现稳定信号，才创建新 family/rig/seed 的预注册 lock；final audit 还要拆成独立进程和事前 decision commit。

### 实验 5：最后才进入神经算子比较

在小模型的 oracle closure、Gate A/B 和新锁定证据都成立之前，不开始用网络掩盖可辨识性问题。以后若进入 FNO、FFNO、DeepONet，必须固定同一数据、相机、噪声、split、训练/调参预算与最终指标，再进行独立公平比较。

## 10. 今日结论（必须保留）

**尚未证明任何新算法优于 FNO、FFNO 或 DeepONet；当前 v5b 也没有进行这三类方法的公平同任务比较。本项目尚未形成可投稿论文。**

> v5a 在规定合成锁定集上暴露了严重的形态/算子混淆与接受尾部风险；v5b M1 已在修正支撑域

> 4/6，当前接受门又让 5/11 个 accepted 样本在独立 audit view 变差。下一轮应检验公平正则选择
> 与外视角风险排序，而不是包装成功。
