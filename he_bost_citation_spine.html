He / BOST 引用脊柱

这页不是再堆论文，而是把何远哲相关路线压成一条可以向师兄解释的引用骨架：从 BOS 位移、BOST 体重构，到 NeRIF 神经隐式折射率场，再到 PIV-BOST 速度补偿和 4D BOST 时序重构。最后更新：2026-07-10。

何远哲同步主线 选题执行手册 Agent 辅助 BOST 14 天精读执行包 论文库 561 篇 NRIP 可复现性审计 TDBOST / 4D BOST 审计 受限全文队列 代码/数据复用
A本科主线：NeRIF/BOST 鲁棒性。
B/C升级线：PIV-BOST 或 4D BOST 子问题。
F工具线：位移估计、质量控制、数据接口。

总判断

本科毕设最稳的写法是：以 NeRIF/BOST 为主线，先复现一个可控的 neural-field / voxel baseline 对比，再用视角数、噪声、mask、位移质量和重投影误差做鲁棒性分析。PIV-BOST 和 4D BOST 不作为一开始的完整复现目标，而是作为“如果师兄给数据或明确需求，就升级”的二级模块。

三条可毕业路线

A: NeRIF/BOST 鲁棒性

推荐主线

用合成 phantom 或 Open BOS 数据做少视角重构，比较 voxel/传统 baseline 与 simplified NeRIF。输出视角数、噪声、mask、容量、重投影误差和中心切片。

B: PIV-BOST 图像层补偿

数据驱动升级

如果师兄给 PIV/BOST 同步数据，就做补偿接口；如果没有，先做 synthetic particle image + refractive distortion + OpenPIV/Farneback/RAFT-lite 对照。

C: 4D BOST 子问题

挑战项

不要写完整 TOG 复现。合理目标是 moving Gaussian/flame phantom、逐帧重构 baseline、低秩时间表示、temporal consistency 和内存/时间报告。

最小阅读顺序

  1. BOS physics: 先读 Goldhahn/Seume 2007 的 sensitivity/accuracy/resolution，拿到实验误差语言。
  2. BOST baseline: 读 UBOST 和 Direct BOST-RBF，知道体素、连续 RBF、神经隐式场分别解决什么。
  3. He mainline: 精读 NeRIF 的 formulation、network、loss、numerical validation 和 real flame experiment。
  4. Competitors: 快读 NeDF 和 NRIP，只抽 sparse-view、encoding、mask、gradient loss、real-data benchmark。
  5. Upgrade: 根据师兄反馈二选一：PIV-BOST 的 image/calibration/vector 层，或 4D BOST 的 rank/time consistency 层。

对师兄的判定问题

问题 决定什么 若答案不明确
本科题目是否应以 NeRIF/BOST 鲁棒性为主？ 是否锁定 A 路线。 先做 A，保留 B/C 接口。
组内认可的传统 baseline 是 UBOST、Direct BOST-RBF、SIRT/Landweber，还是组内代码？ 决定论文比较对象。 先做 voxel toy + NeRIF toy，只把 UBOST/RBF 写作强 baseline。
PIV-BOST 里补偿发生在图像层、互相关层、标定层还是速度矢量层？ 决定 B 路线是否能本科化。 先做 synthetic image-layer toy，不声称完整 PIV-BOST。
六轴与交叉实验已完成后，4D BOST 最有价值的是无真值 rank selector、bias/sampling 阈值、可视化/数据清洗，还是接真实数据？ 决定 C 路线子问题。 用现有 moving-phantom operating map 作为会前证据，不重复 rank 扫描。
是否能给一小份可脱敏样例数据和字段说明？ 决定是否从合成验证升级到真实接口。 继续使用 Open BOS / synthetic manifest，不等待数据才开工。

边界提醒

PIV-BOST、UBOST、Direct BOST-RBF 等受限全文继续走本地私有库或出版社入口；4D BOST 已核 ACM Open access / CC BY 4.0 HTML 与 eReader 可读，但 raw PDF 尚未本地缓存。公开站点只放合法开放文件、官方直读入口和自己的分析。当前公开论文库为 561 篇入口、192 个已校验开放 PDF。NRIP 与 TDBOST 仓库虽然公开，但根目录无 LICENSE，只用于结构阅读和独立 clean-room 实验设计；TDBOST 的正文精读、paper-vs-code 差异、M3B rank 结果与系统偏差边界见专项审计页。Agent for Science 仍只作为 BOST/NeRIF 工作流辅助层。
