选题路线深挖矩阵

把“我可以做什么”拆成可问师兄、可拿数据、可写论文、可降级毕业的路线。核心仍然贴住何远哲的 BOST / NeRIF / PIV-BOST / 4D BOST。最后更新：2026-07-09。

师兄审核入口 选题执行手册 第一月实验执行板 VPN 与私有论文库边界 论文跳转库 F 位移质量控制 学习日志模板

总判断

本科最稳主线不是泛 AI，也不是完整实验系统，而是“BOST/NeRIF 重构闭环 + 数据质量/误差分析 + 可升级到 PIV-BOST 或 4D BOST 的接口”。这个主线能同时满足蔡老师的计算成像/机器学习流动显示方向、何远哲的论文谱系和本科可交付性。

路线优先级

D 实用 BOST/PIV-BOST 真实数据质量控制与 view registration 报告工具 服务师兄真实实验处理。 高：有数据时很有用，无数据时可用模板。 需要组内最小样例和保密边界。 manifest、mask/ROI、view geometry、displacement confidence、bad-view report。
F 工具 BOS/PIV-BOST 位移估计、光流 benchmark 与质量控制工具 直接服务 BOST/NeRIF/PIV-BOST 的图像入口和真实数据筛查。 高：合成背景/粒子图像即可启动。 不能跑成纯 CV 排行榜，必须接回重构或速度误差传播。 OpenPIV/Farneback/DIS/RAFT-PIV 对照、bad-region mask、confidence、runtime、downstream error。
G 辅助 Agent for BOST 参数扫描、日志和图表报告 贴蔡老师 Agent for Science，但不应单独成主线。 中高：作为 A/B/C 的工具层很好。 如果没有物理 pipeline，会变成泛工具。 实验 registry、自动画图、失败样本归档、每周报告生成。
H 旁支 事件相机高速流动/结构可视化的数据重构与质量评估 贴蔡组 event-triggered BOS、adaptive event-frame schlieren、EBIV/DMD，不是何远哲主线。 中：公开论文多，真实 event 数据要问组里。 容易偏成传感器/计算机视觉题，需绑定 BOS/PIV-BOST 图像层。 事件数据表示、time-window reconstruction、frame-event fusion toy、重建质量指标。
I 备选 单相机/金属颗粒多物理量测量的图像算法复现与误差分析 贴 OERF 低碳燃料、金属颗粒燃烧和计算成像，不贴 He/BOST 主线。 中：可做图像算法复现，但实验数据依赖强。 物理背景和燃烧化学会增多，可能冲淡 BOST 主线。 RGB thermometry/sizing toy、粒子检测、尺寸/温度误差传播和数据需求清单。

旁支路线怎么用

H：事件相机高速可视化

  • 代表文献：event-triggered BOS、adaptive event-frame schlieren、EBIV/DMD benchmark、flexible-structure event-triggered visualization。
  • 可迁移到主线的能力：高频采样、低存储压力、时间窗重构、frame-event fusion、DMD/投影背景设计。
  • 本科可问法：如果组里有 event camera 数据，我能否做一个“事件数据到 BOS/PIV 位移质量指标”的工具，而不是完整传感器论文？

I：金属颗粒/单相机诊断

  • 代表文献：iron particle heat exchange、single RGB camera thermometry/sizing、digital in-line holography、iron/ammonia co-firing。
  • 可迁移到主线的能力：低成本成像、多物理量同步测量、粒子检测、温度/尺寸误差传播、实验数据标注。
  • 本科可问法：如果 BOST 数据暂时不给，能否拿一小段公开或组内粒子图像做图像测量算法复现，作为备选毕业题？

A 路线：NeRIF/BOST 主线拆解

物理问题

  • 折射率场或密度场如何导致背景图案位移。
  • 多视角位移/偏折观测如何反推 3D refractive index field。
  • 少视角、噪声、遮挡、mask、相机几何误差如何影响重构。

算法问题

  • 体素网格、SART/TV/LSQR 与 coordinate MLP 的差异。
  • Fourier/SIREN/hash encoding 对高频折射率结构的影响。
  • loss 如何同时包含 projection consistency、smoothness 和边界约束。

论文贡献写法

  • 建立可复现实验闭环和 baseline。
  • 系统比较 view count、噪声、网络容量、encoding、regularization。
  • 给出面向真实 BOST 数据接入的质量控制和参数建议。

B 路线：PIV-BOST 降级与升级

从 Springer 摘要能先做出的拆解

2025 PIV-BOST 论文的正式摘要已经足够把本科 toy 定得更具体：用一组多视角/内窥式 BOST 重构火焰周围折射率场，再把折射率梯度转成 PIV 平面上的像素偏移，最后评估速度误差补偿。你不需要等全文才开始写代码；先做 synthetic refractive-index field、particle image pair、OpenPIV 互相关和 before/after velocity error。

层级 能做什么 数据需求 问师兄的问题
图像层 模拟折射造成的粒子图像偏折，比较 dewarping 前后互相关峰和速度误差。 synthetic PIV image 或 OpenPIV/piv-image-generator。 组里 PIV-BOST 补偿是先修正图像，还是先重构折射率场再修正速度？
标定层 把 refractive distortion 当作 disparity / calibration error，做非参数校正 toy。 相机标定、视角几何、背景/粒子图像。 stereo PIV-BOST 的核心痛点是 calibration、matching、还是 vector correction？
速度层 比较补偿前后速度矢量场的 bias、RMSE、vorticity、divergence 和异常向量比例。 PIV 速度场或可公开 synthetic velocity field。 师兄希望本科交付图像工具、速度误差图，还是完整 PIV-BOST pipeline 的一个模块？

F 路线：位移/光流质量控制，不做纯 CV 排行榜

最稳的工具型交付

如果师兄说真实 BOST/PIV-BOST 最大痛点在图像前处理、位移场、坏区域或筛帧，F 路线可以直接启动。它用 synthetic background / particle image pair 起步，比较 OpenPIV、Farneback、DIS/TV-L1 或 RAFT-lite，输出 bad-region mask、confidence、runtime 和 downstream error，并把结果接回 A 的 NeRIF/BOST 重构、B 的 PIV-BOST 补偿或 D 的真实数据质量报告。详细执行页见 displacement_qc_route_f.md。

C 路线：4D BOST 不完整复现，只拆可毕业子问题

可做子问题

  • 动态 phantom：平移涡、脉动热羽流、moving Gaussian density blob。
  • 逐帧重构 vs 低秩/张量表示：比较时间一致性和噪声传播。
  • rank/noise/frame-count/view-count 扫描：形成毕业论文中最扎实的结果图。
  • DMD mode energy、low-rank+sparse residual、temporal smoothness 作为评价指标。

不要承诺

  • 不承诺复现 ACM TOG 全部网络和实验结果。
  • 不承诺拿到完整高速多视角真实数据。
  • 不把 TDBOST 无许可证仓库代码直接搬进本科项目。
  • 不把 4D 论文级画质作为第一阶段目标。

  • 第一月你最希望我交付 data loader、baseline、参数扫描还是可视化报告？

  • 是否能给一份最小不涉密样例：flow-off、flow-on、mask、视角几何、位移场？
  • 哪些图可以写进本科论文，哪些只能内部展示？

首选 A 路线工作包

工作包 目标 具体任务 输出图表 失败降级
WP1 forward model 把 BOST 观测链条从概念变成可运行代码。 构造 3D refractive-index phantom；近似生成多视角 projection / displacement；保留相机数、噪声、mask、ROI 参数。 phantom 切片、view-wise displacement、重投影示意图。 如果 3D 太重，先做 2D Radon / projected-density toy，再逐步升维。
WP2 classical baseline 避免毕业设计变成“只训练一个网络”。 实现或调用 FBP/SART/LSQR/Tikhonov 至少一种；统一输入输出和误差指标。 baseline 重构切片、error map、view-count 曲线。 如果 3D baseline 难写，先用 scikit-image 2D SART 建立可解释对照。
WP3 neural field 做简化 NeRIF，不追求论文全量复现。 coordinate MLP 输入 `(x,y,z)` 输出 `n` 或 `delta n`；比较 Fourier / SIREN / plain MLP；加入 projection loss 与 smoothness。 loss curve、切片对比、encoding 消融、训练时间表。 如果 ray integration 过重，先用 projection-matrix toy 或 precomputed rays。
WP4 robustness map 形成最像研究贡献的结果。 扫描 view count、noise、mask ratio、bad view、network width、encoding frequency；比较 neural vs classical。 heatmap、pareto 曲线、失败案例 gallery、推荐参数表。 如果 GPU 不够，把扫描维度减少到 view/noise/encoding 三项。
WP5 real-data adapter 给组内真实数据留接口，即使暂时拿不到数据也不空。 定义 manifest；检查 flow-off / flow-on / geometry / displacement / mask / permissions；能加载一组样例并画 view-quality report。 数据完整性表、view quality 图、bad-view 标记、缺失字段列表。 没有真实数据时用 open BOS / synthetic manifest 演示完整流程。

  • 收束论文：方法、实验、讨论、失败边界和未来工作。
  • 准备答辩：把“我没完整复现论文”转化为“我完成了可复现子问题和误差图谱”。

最值得避免的坑

技术坑

  • 一上来追完整 4D BOST 或真实实验台，导致三个月没有可运行结果。
  • 只调神经网络，不写 forward model 和 baseline，论文站不住。
  • 只报平均 L2，不看局部失败、bad view、mask、重投影误差和训练时间。
  • 把 PIV-BOST 写成速度场后处理，却没有说明图像层、标定层、速度层的接口区别。

沟通坑

  • 问师兄“能不能给数据”太泛；应问具体字段、单位、权限和第一周交付。
  • 把课题组所有方向都当候选主线，导致导师看不到你真正要做什么。
  • 公开网页上传订阅 PDF；正确做法是公开页放 DOI/开放 PDF，订阅版只放本地私有库。
  • 学习流体力学没有产物；每周必须把一个概念转成图、代码或问题。
本页是选题决策矩阵，不替代师兄确认。每条路线都保留“最小可毕业版本”和“数据到位后的升级版本”。
